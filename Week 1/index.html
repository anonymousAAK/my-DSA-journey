👋 Hello World -- Your First Program

Every Java program must have at least one class and a main() method, which serves as the entry point for execution. The classic "Hello World" program introduces this basic structure.

Key Concepts

  • Every Java file must contain a class definition
  • The main() method is the entry point: public static void main(String args[])
  • System.out.println() prints text followed by a newline
  • Java is case-sensitive: System is not the same as system
  • Every statement ends with a semicolon (;)

Implementation

hello_world.java
package fundamentals; public class HelloWorld { public static void main(String args[]) { System.out.println("Hello World"); } } // public static void main(String[] args) // 1. This is the line at which the program will begin executing. // 2. The curly braces {} indicate start and end of main.
fundamentals.cpp
#include <iostream> using namespace std; int main() { cout << "Hello, World!" << endl; return 0; }
fundamentals.py
# Python: no class or main method required! print("Hello, World!")
fundamentals.rs
fn main() { println!("Hello, World!"); }
Key Insight
In Java, the file name should match the public class name. The main method signature must be exactly public static void main(String[] args) for the JVM to find and execute it.

📦 Variables & Addition

A variable is the basic unit of storage in Java. Variables must be declared with a type before use, and local variables are not assigned default values by the compiler.

Variable Naming Rules

  • May contain letters (a-z, A-Z), digits (0-9), underscore (_), and dollar sign ($)
  • Must not begin with a digit
  • Java is case-sensitive: myVar and MyVar are different
  • Cannot be a reserved keyword (e.g., int, class, void)

Implementation

Add_two_numbers.java
package fundamentals; public class AddTwoNumbers { public static void main(String args[]) { int a = 10; // Declare and initialize first operand int b = 25; // Declare and initialize second operand int c = a + b; // Store the sum in a new variable System.out.println(c); // Output: 35 } }
fundamentals.cpp
int main() { int a = 10; int b = 25; int c = a + b; cout << c << endl; // 35 // C++ also has 'auto' for type inference auto score = 95.5; // compiler deduces: double return 0; }
fundamentals.py
# Python: no type declaration needed (dynamically typed) a = 10 b = 25 c = a + b print(c) # 35 # Variables can change type at any time a = "now I'm a string"
fundamentals.rs
fn main() { let a = 10; // immutable by default let b = 25; let c = a + b; println!("{}", c); // 35 let mut x = 5; // mut = mutable x += 1; // allowed because x is mut println!("x = {}", x); }

🔢 Arithmetic Operations

Java supports standard arithmetic operators. A critical pitfall: integer division truncates the result -- 10/6 gives 1, not 1.666.

Operators

OperatorOperationExampleResult
+Addition10 + 313
-Subtraction10 - 37
*Multiplication10 * 330
/Division10 / 33 (integer!)
%Modulus10 % 31

Implementation

Arthemetic_Operation.java
package fundamentals; public class ArithematicOperators { public static void main(String args[]) { int a = 3; int b = 10; int c = b / (2 * a); // 10/6 = 1 (integer division truncates) System.out.println(c); // Output: 1 } }
fundamentals.cpp
int a = 17, b = 5; cout << "a + b = " << (a + b) << endl; // 22 cout << "a - b = " << (a - b) << endl; // 12 cout << "a * b = " << (a * b) << endl; // 85 cout << "a / b = " << (a / b) << endl; // 3 (integer division!) cout << "a % b = " << (a % b) << endl; // 2
fundamentals.py
a, b = 17, 5 print(f"a / b = {a / b}") # 3.4 (true division -- always float!) print(f"a // b = {a // b}") # 3 (floor division -- integer result) print(f"a % b = {a % b}") # 2 (modulo) print(f"a ** b = {a ** b}") # 1419857 (power -- no Math.pow needed)
fundamentals.rs
let a = 17_i32; let b = 5_i32; println!("a + b = {}", a + b); // 22 println!("a / b = {}", a / b); // 3 (integer division) println!("a % b = {}", a % b); // 2 // Rust has no ** operator; use .pow() or f64::powi() println!("a^b = {}", (a as f64).powi(b)); // 1419857
Common Pitfall
Integer division in Java, C++, and Rust truncates the decimal part. To get a precise result, cast one operand to double/float: (double)a / b or use 3.0 instead of 3.

📊 Data Types

Java has 8 primitive data types. The OS allocates memory based on the declared type, and each type determines what values can be stored.

Primitive Types Overview

Integer Types: byte 1 byte [-128, 127] short 2 bytes [-32,768, 32,767] int 4 bytes [-2^31, 2^31 - 1] (~2.1 billion) long 8 bytes [-2^63, 2^63 - 1] Floating Point: float 4 bytes ~7 decimal digits (suffix 'f' required: 3.14f) double 8 bytes ~15 decimal digits (default for decimals) Other: char 2 bytes Unicode character (Java uses Unicode, not ASCII!) boolean ? true / false (size not precisely defined) Note: String is NOT a primitive -- it is a reference type (class).

Reading Different Types with Scanner

Data_Types.java
import java.util.Scanner; public class DataTypes { public static void main(String args[]) { Scanner s = new Scanner(System.in); String str = s.next(); // Read a single word/token System.out.print(str); // print() -- no newline // vs System.out.println() -- adds newline s.nextInt(); // Read int s.nextDouble(); // Read double s.nextLong(); // Read long s.nextLine(); // Read entire line } }
fundamentals.cpp
cout << "char: " << sizeof(char) << " byte(s)" << endl; // 1 cout << "short: " << sizeof(short) << " byte(s)" << endl; // 2 cout << "int: " << sizeof(int) << " byte(s)" << endl; // 4 cout << "long long: " << sizeof(long long) << " byte(s)" << endl; // 8 cout << "float: " << sizeof(float) << " byte(s)" << endl; // 4 cout << "double: " << sizeof(double) << " byte(s)" << endl; // 8 cout << "bool: " << sizeof(bool) << " byte(s)" << endl; // 1
fundamentals.py
# Python: dynamically typed -- no byte, short, long # int handles ALL integer sizes (arbitrary precision!) big = 2 ** 100 # Works! Java would overflow with int/long. print(f"Type of 42: {type(42)}") # <class 'int'> print(f"Type of 3.14: {type(3.14)}") # <class 'float'> print(f"Type of 'hi': {type('hi')}") # <class 'str'> print(f"Type of True: {type(True)}") # <class 'bool'>
fundamentals.rs
// Rust has explicit signed/unsigned integer types println!("i8 range: [{}, {}]", i8::MIN, i8::MAX); println!("i32 range: [{}, {}]", i32::MIN, i32::MAX); println!("i64 range: [{}, {}]", i64::MIN, i64::MAX); println!("f64 max: {}", f64::MAX); println!("char size: {} bytes", std::mem::size_of::<char>()); // 4 bytes! // Rust char is 4 bytes (full Unicode scalar value)

⌨ Taking User Input

The Scanner class reads user input from the console. It breaks input into tokens using whitespace as a delimiter and provides methods to parse various primitive types.

Scanner Methods

MethodReadsNotes
nextInt()IntegerReads next token as int
nextDouble()DoubleReads next token as double
nextLong()LongReads next token as long
next()String (word)Stops at whitespace
nextLine()String (line)Reads entire remaining line

Implementation

Taking_Input.java
import java.util.Scanner; public class TakingInput { public static void main(String args[]) { Scanner s = new Scanner(System.in); int a = s.nextInt(); // Read first integer int b = s.nextInt(); // Read second integer int c = a + b; String str = s.nextLine(); // Read remaining text on line char ch = str.charAt(0); // Extract first character System.out.println(ch); } }
fundamentals.cpp
#include <iostream> using namespace std; int main() { int num; cout << "Enter a number: "; cin >> num; cout << "Double of " << num << " is " << num * 2 << endl; return 0; }
fundamentals.py
# input() always returns a string -- cast as needed name = input("Enter your name: ") print(f"Hello, {name}!") num = int(input("Enter a number: ")) print(f"Double of {num} is {num * 2}")
fundamentals.rs
use std::io; fn main() { println!("Enter a number:"); let mut input = String::new(); io::stdin().read_line(&mut input).unwrap(); let num: i32 = input.trim().parse().unwrap(); println!("Double of {} is {}", num, num * 2); }
Common Pitfall
Calling nextLine() after nextInt() may read the leftover newline character from the integer input. Solution: add an extra s.nextLine() call to consume the dangling newline before reading the actual line.

🔄 Typecasting

Typecasting converts a value from one data type to another. Java supports two kinds: widening (automatic) and narrowing (explicit).

Conversion Hierarchy

Widening (Automatic): smaller type → larger type, NO data loss byte → short → int → long → float → double Narrowing (Explicit): larger type → smaller type, MAY lose data Requires cast operator: (targetType) value Example: (long) 100.99 = 100 (truncates, does NOT round!)

Implementation

Typecasting.java
public static void main(String[] args) { int i = 100; long l1 = i; // Widening: int -> long (automatic) double d = 100.04; long l2 = (long) d; // Narrowing: double -> long (explicit, truncates) System.out.println(i); // 100 System.out.println(l1); // 100 System.out.println(d); // 100.04 System.out.println(l2); // 100 (decimal part lost!) }
fundamentals.cpp
double pi = 3.14159; int truncated = (int)pi; // C-style cast: 3 int truncated2 = static_cast<int>(pi); // C++ style (preferred) double promoted = (double)25; // int -> double cout << "(int)3.14159 = " << truncated << endl; // 3 cout << "5 / 2 = " << (5 / 2) << endl; // 2 (int division) cout << "5.0 / 2 = " << (5.0 / 2) << endl; // 2.5 (float division)
fundamentals.py
# Python: explicit casting with type constructors x = int(3.7) # 3 (truncates, not rounds) y = float(5) # 5.0 z = str(42) # "42" w = int("123") # 123 print(f"int(3.7) = {x}, float(5) = {y}")
fundamentals.rs
let pi: f64 = 3.14159; let truncated = pi as i32; // 3 (truncation, not rounding) let promoted = 25_i32 as f64; // 25.0 println!("3.14159 as i32 = {}", truncated); // 3 println!("5 / 2 = {}", 5 / 2); // 2 (integer division) println!("5.0 / 2.0 = {}", 5.0_f64 / 2.0); // 2.5

⚙ Operators in Java

Java provides a rich set of operators for performing operations on variables and values.

Average of Two Numbers -- Practical Example

Average_of_two_numbers.java
import java.util.Scanner; public class Average { public static void main(String[] args) { Scanner sc = new Scanner(System.in); String name = sc.next(); // Read the user's name int a = sc.nextInt(); // Read first number int b = sc.nextInt(); // Read second number int c = sc.nextInt(); // Read third number System.out.println(name); System.out.println((a + b + c) / 3); // Integer division: result is truncated, not rounded // Use /3.0 for precise average } }
fundamentals.cpp
// Comparison operators cout << boolalpha; // print true/false instead of 1/0 cout << "5 == 5: " << (5 == 5) << endl; // true cout << "5 != 3: " << (5 != 3) << endl; // true cout << "5 > 3: " << (5 > 3) << endl; // true // Logical operators cout << "true && false: " << (true && false) << endl; // false cout << "true || false: " << (true || false) << endl; // true cout << "!true: " << (!true) << endl; // false
fundamentals.py
# Comparison print(f"5 == 5: {5 == 5}") # True print(f"5 != 3: {5 != 3}") # True # Logical -- Python uses words, not symbols print(f"True and False: {True and False}") # False print(f"True or False: {True or False}") # True print(f"not True: {not True}") # False
fundamentals.rs
println!("5 == 5: {}", 5 == 5); // true println!("5 != 3: {}", 5 != 3); // true println!("true && false: {}", true && false); // false println!("true || false: {}", true || false); // true println!("!true: {}", !true); // false
Key Insight
Parentheses around (a + b + c) are essential for the average calculation. Without them, only c would be divided by 3 due to operator precedence (multiplication/division before addition).
