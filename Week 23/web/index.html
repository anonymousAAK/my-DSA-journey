TSP with Bitmask DP

The Traveling Salesman Problem asks for the shortest tour visiting all cities exactly once and returning to the start. The brute-force approach checks n! permutations, but bitmask DP reduces this to O(n^2 * 2^n) by representing visited cities as bits in an integer.

State Representation

dp[mask][i] = minimum cost to reach city i having visited exactly the cities in mask

Where mask is a bitmask: bit j is set if city j has been visited. For n=4 cities, mask=0b1011 means cities 0, 1, and 3 have been visited.

Bitmask State Space for n=4 cities:

mask=0001 (city 0 only)     mask=0011 (cities 0,1)
  dp[0001][0] = 0             dp[0011][1] = dist[0][1]

mask=0101 (cities 0,2)      mask=0111 (cities 0,1,2)
  dp[0101][2] = dist[0][2]    dp[0111][1] = min of paths ending at 1
                                dp[0111][2] = min of paths ending at 2

mask=1111 (all cities)       Answer: min(dp[1111][i] + dist[i][0])
  dp[1111][1] = best tour       for all i != 0
  dp[1111][2] = ending at 2
  dp[1111][3] = ending at 3

Total states: 2^n * n = 2^4 * 4 = 64 (vs 4! = 24 permutations)
For n=20: 2^20 * 20 ~ 20M states (vs 20! ~ 2.4 * 10^18)

Bitmask DP trades permutation enumeration (n!) for subset enumeration (2^n), enabling practical solutions for n up to ~20.

Key Insight

The key observation is that the optimal tour through a set of cities depends only on which cities have been visited and where we currently are -- not the order in which previous cities were visited. This "optimal substructure over subsets" is what makes bitmask DP work. The same principle applies to the Hamiltonian Path problem, assignment problems, and scheduling with dependencies.

// TSP with Bitmask DP (Java)
static int tspMinCost(int[][] dist) {
    int n = dist.length;
    int FULL = (1 << n) - 1;
    int[][] dp = new int[1 << n][n];
    for (int[] row : dp) Arrays.fill(row, Integer.MAX_VALUE / 2);
    dp[1][0] = 0; // start at city 0

    for (int mask = 1; mask <= FULL; mask++) {
        for (int u = 0; u < n; u++) {
            if ((mask & (1 << u)) == 0 ||
                dp[mask][u] == Integer.MAX_VALUE / 2) continue;
            for (int v = 0; v < n; v++) {
                if ((mask & (1 << v)) != 0) continue;
                int newMask = mask | (1 << v);
                dp[newMask][v] = Math.min(dp[newMask][v],
                    dp[mask][u] + dist[u][v]);
            }
        }
    }
    int ans = Integer.MAX_VALUE;
    for (int u = 1; u < n; u++)
        if (dp[FULL][u] != Integer.MAX_VALUE / 2)
            ans = Math.min(ans, dp[FULL][u] + dist[u][0]);
    return ans;
}
// tspMinCost(4-city symmetric) = 80
// TSP — bitmask DP (C++17)
int tsp_bitmask(const std::vector<std::vector<int>>& dist) {
    int n = dist.size();
    int full = (1 << n) - 1;
    std::vector dp(1 << n, std::vector<int>(n, INT_MAX));
    dp[1][0] = 0;

    for (int mask = 1; mask <= full; ++mask)
        for (int u = 0; u < n; ++u) {
            if (dp[mask][u] == INT_MAX) continue;
            if (!(mask & (1 << u))) continue;
            for (int v = 0; v < n; ++v) {
                if (mask & (1 << v)) continue;
                int nm = mask | (1 << v);
                dp[nm][v] = std::min(dp[nm][v],
                    dp[mask][u] + dist[u][v]);
            }
        }
    int ans = INT_MAX;
    for (int u = 1; u < n; ++u)
        if (dp[full][u] != INT_MAX)
            ans = std::min(ans, dp[full][u] + dist[u][0]);
    return ans;
}
# TSP with Bitmask DP + path reconstruction (Python)
def tsp_bitmask(dist):
    n = len(dist)
    full = (1 << n) - 1
    INF = float('inf')
    dp = [[INF] * n for _ in range(1 << n)]
    parent = [[-1] * n for _ in range(1 << n)]
    dp[1][0] = 0

    for mask in range(1 << n):
        for u in range(n):
            if dp[mask][u] == INF: continue
            if not (mask & (1 << u)): continue
            for v in range(n):
                if mask & (1 << v): continue
                new_mask = mask | (1 << v)
                new_cost = dp[mask][u] + dist[u][v]
                if new_cost < dp[new_mask][v]:
                    dp[new_mask][v] = new_cost
                    parent[new_mask][v] = u

    min_cost = INF
    last_city = -1
    for u in range(n):
        total = dp[full][u] + dist[u][0]
        if total < min_cost:
            min_cost = total
            last_city = u
    return min_cost
// TSP Bitmask DP (Rust)
fn tsp_bitmask(dist: &[Vec<i64>]) -> i64 {
    let n = dist.len();
    let full = (1 << n) - 1;
    let inf = i64::MAX / 2;
    let mut dp = vec![vec![inf; n]; 1 << n];
    dp[1][0] = 0;

    for mask in 1..(1 << n) {
        for u in 0..n {
            if dp[mask][u] >= inf { continue; }
            if mask & (1 << u) == 0 { continue; }
            for v in 0..n {
                if mask & (1 << v) != 0 { continue; }
                let nm = mask | (1 << v);
                let cost = dp[mask][u] + dist[u][v];
                if cost < dp[nm][v] { dp[nm][v] = cost; }
            }
        }
    }
    (0..n).map(|u| dp[full][u] + dist[u][0])
          .min().unwrap_or(inf)
}

Optimal LIS: O(n log n)

The classic O(n^2) LIS algorithm checks all previous elements. The optimal approach uses patience sorting: maintain an array tails where tails[i] is the smallest tail element of any increasing subsequence of length i+1. Binary search on this sorted array gives O(n log n).

Why "Patience Sorting"?

Named after the card game Patience (Solitaire). Imagine placing cards left to right: each card goes on the leftmost pile whose top card is >= the current card, or starts a new pile. The number of piles equals the LIS length.

Key Insight

The tails array is always sorted. For each new element, use binary search (lower_bound) to find where it fits. If it extends the longest subsequence (goes past the end), append it. Otherwise, replace the first tail >= it. This keeps tails as small as possible, maximizing future extension potential.

// LIS O(n log n) — patience sorting (Java)
static int lisOptimal(int[] nums) {
    int[] tails = new int[nums.length];
    int size = 0;
    for (int x : nums) {
        int lo = 0, hi = size;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (tails[mid] < x) lo = mid + 1;
            else hi = mid;
        }
        tails[lo] = x;
        if (lo == size) size++;
    }
    return size;
}
// [10,9,2,5,3,7,101,18] -> LIS = 4
// LIS O(n log n) with lower_bound (C++17)
int lis_nlogn(const std::vector<int>& nums) {
    std::vector<int> tails;
    for (int x : nums) {
        auto it = std::lower_bound(
            tails.begin(), tails.end(), x);
        if (it == tails.end()) tails.push_back(x);
        else *it = x;
    }
    return tails.size();
}
# LIS O(n log n) with path reconstruction (Python)
def lis_with_reconstruction(nums):
    if not nums: return []
    tails, tails_idx = [], []
    predecessor = [-1] * len(nums)

    for i, num in enumerate(nums):
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
            tails_idx.append(i)
        else:
            tails[pos] = num
            tails_idx[pos] = i
        if pos > 0:
            predecessor[i] = tails_idx[pos - 1]

    # Reconstruct the actual LIS
    lis = []
    idx = tails_idx[-1]
    while idx != -1:
        lis.append(nums[idx])
        idx = predecessor[idx]
    lis.reverse()
    return lis
// LIS O(n log n) — patience sorting (Rust)
fn lis_optimal(nums: &[i32]) -> usize {
    let mut tails: Vec<i32> = Vec::new();
    for &num in nums {
        let pos = tails.partition_point(|&x| x < num);
        if pos == tails.len() {
            tails.push(num);
        } else {
            tails[pos] = num;
        }
    }
    tails.len()
}

Matrix Chain Multiplication

Given matrices A1 (p0 x p1), A2 (p1 x p2), ..., An (p(n-1) x pn), find the parenthesization that minimizes total scalar multiplications. This is an interval DP problem.

dp[i][j] = min cost to multiply matrices i through j
dp[i][j] = min over k in [i, j-1] of: dp[i][k] + dp[k+1][j] + p[i-1] * p[k] * p[j]
Example: dims = [10, 30, 5, 60]  (3 matrices)
  A1: 10x30,  A2: 30x5,  A3: 5x60

Option 1: (A1*A2)*A3 = 10*30*5 + 10*5*60 = 1500 + 3000 = 4500
Option 2: A1*(A2*A3) = 30*5*60 + 10*30*60 = 9000 + 18000 = 27000

DP Table:         j=1    j=2    j=3
           i=1  [  0    1500   4500 ]
           i=2  [  -      0    9000 ]
           i=3  [  -      -       0 ]

Optimal: 4500 with parenthesization (A1 x A2) x A3

The order of multiplication matters enormously: 4500 vs 27000 scalar multiplications.

// Matrix Chain Multiplication (C++17)
long long matrix_chain_mult(const std::vector<int>& p) {
    int n = p.size() - 1;
    if (n <= 1) return 0;
    std::vector dp(n, std::vector<long long>(n, 0));

    for (int l = 2; l <= n; ++l)         // chain length
        for (int i = 0; i <= n - l; ++i) {
            int j = i + l - 1;
            dp[i][j] = LLONG_MAX;
            for (int k = i; k < j; ++k) {
                long long cost = dp[i][k] + dp[k+1][j]
                    + (long long)p[i] * p[k+1] * p[j+1];
                dp[i][j] = std::min(dp[i][j], cost);
            }
        }
    return dp[0][n - 1];
}
// dims=[10,30,5,60] -> 4500
// dims=[40,20,30,10,30] -> 26000
# Matrix Chain with optimal parenthesization (Python)
def matrix_chain_multiplication(dims):
    n = len(dims) - 1
    if n <= 0: return 0, ""
    dp = [[0] * (n + 1) for _ in range(n + 1)]
    split = [[0] * (n + 1) for _ in range(n + 1)]

    for length in range(2, n + 1):
        for i in range(1, n - length + 2):
            j = i + length - 1
            dp[i][j] = float('inf')
            for k in range(i, j):
                cost = (dp[i][k] + dp[k+1][j]
                      + dims[i-1] * dims[k] * dims[j])
                if cost < dp[i][j]:
                    dp[i][j] = cost
                    split[i][j] = k

    def build_parens(i, j):
        if i == j: return f"A{i}"
        k = split[i][j]
        return f"({build_parens(i,k)} x {build_parens(k+1,j)})"

    return dp[1][n], build_parens(1, n)
# dims=[30,35,15,5,10,20,25] -> 15125
// Matrix Chain Multiplication (Rust)
fn matrix_chain_multiplication(dims: &[i64]) -> i64 {
    let n = dims.len() - 1;
    if n <= 1 { return 0; }
    let mut dp = vec![vec![0i64; n + 1]; n + 1];

    for l in 2..=n {
        for i in 1..=n - l + 1 {
            let j = i + l - 1;
            dp[i][j] = i64::MAX;
            for k in i..j {
                let cost = dp[i][k] + dp[k+1][j]
                    + dims[i-1] * dims[k] * dims[j];
                if cost < dp[i][j] { dp[i][j] = cost; }
            }
        }
    }
    dp[1][n]
}

Digit DP

Digit DP counts numbers in a range [1, N] that satisfy some digit-level constraint. It processes digits from most significant to least, maintaining a tight flag that indicates whether we're still bounded by N's prefix.

The "Tight" Constraint

When building a number digit by digit: if all digits so far exactly match N's prefix, the next digit is bounded by N's next digit (tight=true). Otherwise, any digit 0-9 is allowed (tight=false). Once tight becomes false, it stays false.

Digit DP State Transitions for N=352, counting numbers without digit 4:

Position 0 (hundreds):
  tight=true:  d can be 0..3
    d=0: tight=false (leading zero)
    d=1: tight=false, started
    d=2: tight=false, started
    d=3: tight=true, started   (matches N's digit)

Position 1 (tens), after d=3 (tight=true):
  d can be 0..5  (bounded by N[1]=5)
    d=4: SKIP (contains digit 4)
    d=0,1,2,3,5: valid

Position 1, after d=1 (tight=false):
  d can be 0..9  (any digit)
    d=4: SKIP
    d=0..3,5..9: valid

Memoize on: (position, tight)
Total states: D * 2 where D = number of digits

Digit DP processes each digit position with a tight constraint, branching into all valid choices.

// Digit DP: Count numbers in [1..N] without digit 4 (Java)
static int digitDP(String num, int pos,
                    boolean tight, boolean started) {
    if (pos == num.length())
        return started ? 1 : 0;

    int limit = tight ? (num.charAt(pos) - '0') : 9;
    int count = 0;
    for (int d = 0; d <= limit; d++) {
        if (d == 4) continue; // skip digit 4
        count += digitDP(num, pos + 1,
            tight && (d == limit),
            started || (d != 0));
    }
    return count;
}
// countNoFour(10) = 9, countNoFour(100) = 81
# Digit DP: count without digit 4 (Python)
def count_without_digit_4(n):
    if n <= 0: return 0
    digits = [int(c) for c in str(n)]

    @lru_cache(maxsize=None)
    def dp(pos, tight, started):
        if pos == len(digits):
            return 1 if started else 0
        upper = digits[pos] if tight else 9
        count = 0
        for d in range(0, upper + 1):
            if d == 4: continue
            count += dp(pos + 1,
                       tight and (d == upper),
                       started or (d != 0))
        return count

    return dp(0, True, False)
// Digit DP: no digit 4 in [1, N] (Rust)
fn digit_dp_no_four(n: u64) -> u64 {
    if n == 0 { return 0; }
    let digits: Vec<u32> = n.to_string()
        .chars().map(|c| c.to_digit(10).unwrap())
        .collect();
    let len = digits.len();
    let mut memo = vec![vec![-1i64; 2]; len];

    fn solve(pos: usize, tight: bool,
             digits: &[u32],
             memo: &mut Vec<Vec<i64>>) -> i64 {
        if pos == digits.len() { return 1; }
        let ti = if tight { 1 } else { 0 };
        if memo[pos][ti] != -1 { return memo[pos][ti]; }
        let upper = if tight { digits[pos] } else { 9 };
        let mut count = 0i64;
        for d in 0..=upper {
            if d == 4 { continue; }
            count += solve(pos+1, tight && d==upper,
                          digits, memo);
        }
        memo[pos][ti] = count;
        count
    }
    (solve(0, true, &digits, &mut memo) - 1) as u64
}

DP on Trees: Maximum Independent Set

A Maximum Independent Set on a tree selects vertices with maximum total weight such that no two selected vertices are adjacent. This uses post-order DFS to compute DP values bottom-up.

dp[v][0] = max weight if v is NOT selected = sum(max(dp[child][0], dp[child][1]))
dp[v][1] = max weight if v IS selected = weight[v] + sum(dp[child][0])
Key Insight

Tree DP always follows the same pattern: process children first (post-order), then combine results at the parent. The tree structure guarantees no cycles, so each node's state depends only on its subtree. This makes tree DP O(n) -- each edge is traversed once.

// Max Independent Set on Tree (Java)
static void dfsTree(int node, int[] weight,
    int[][] children, int[][] dp, boolean[] visited) {
    visited[node] = true;
    dp[node][1] = weight[node]; // take this node
    dp[node][0] = 0;             // don't take
    for (int child : children[node]) {
        if (!visited[child]) {
            dfsTree(child, weight, children, dp, visited);
            // if we take node, children can't be taken
            dp[node][1] += dp[child][0];
            // if we don't take, children go either way
            dp[node][0] += Math.max(
                dp[child][0], dp[child][1]);
        }
    }
}
// Answer: max(dp[root][0], dp[root][1])
// Max Independent Set on Tree (C++17)
int max_independent_set(
    const std::vector<std::vector<int>>& adj,
    const std::vector<int>& weights) {
    int n = adj.size();
    std::vector<std::array<int, 2>> dp(n, {0, 0});
    std::vector<bool> visited(n, false);

    std::function<void(int)> dfs = [&](int u) {
        visited[u] = true;
        dp[u][1] = weights[u];
        for (int v : adj[u]) {
            if (visited[v]) continue;
            dfs(v);
            dp[u][0] += std::max(dp[v][0], dp[v][1]);
            dp[u][1] += dp[v][0];
        }
    };
    dfs(0);
    return std::max(dp[0][0], dp[0][1]);
}
# Max Independent Set — iterative post-order (Python)
def max_independent_set_tree(n, edges, weights):
    adj = defaultdict(list)
    for u, v in edges:
        adj[u].append(v)
        adj[v].append(u)

    visited = [False] * n
    parent_map = [-1] * n
    order = []
    stack = [0]
    visited[0] = True
    while stack:
        node = stack.pop()
        order.append(node)
        for nb in adj[node]:
            if not visited[nb]:
                visited[nb] = True
                parent_map[nb] = node
                stack.append(nb)

    dp_not = [0] * n
    dp_sel = [0] * n
    for v in reversed(order):
        dp_sel[v] = weights[v]
        for child in adj[v]:
            if child == parent_map[v]: continue
            dp_sel[v] += dp_not[child]
            dp_not[v] += max(dp_not[child], dp_sel[child])
    return max(dp_not[0], dp_sel[0])

Summary & Cheat Sheet

Bitmask DP (TSP)

O(n^2 * 2^n). Represent subsets as integers. Feasible for n <= 20. Used for Hamiltonian paths, assignment, scheduling.

LIS O(n log n)

Patience sorting with binary search. tails array stays sorted. lower_bound determines position.

Matrix Chain Mult

O(n^3) interval DP. dp[i][j] = min cost for matrices i..j. Try all split points k.

Digit DP

Count numbers by constraints. State: (position, tight, ...custom). Process digits left to right.

Tree DP

Post-order DFS. dp[node][taken/not] pattern. O(n). Used for independent set, coloring, matching.
