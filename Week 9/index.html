🔮 Bubble Sort

Bubble Sort repeatedly walks through the list, compares adjacent elements, and swaps them if they are out of order. After each pass, the largest unsorted element "bubbles up" to its correct position at the end.

Real-World Analogy

Imagine sorting a hand of playing cards by repeatedly comparing two adjacent cards and swapping if they are out of order. After one full pass, the largest card ends up on the right. Repeat until no swaps are needed.

Step-by-Step Visualization

Array: [64, 25, 12, 22, 11] Pass 1: [64, 25, 12, 22, 11] swap 64 > 25 → [25, 64, 12, 22, 11] [25, 64, 12, 22, 11] swap 64 > 12 → [25, 12, 64, 22, 11] [25, 12, 64, 22, 11] swap 64 > 22 → [25, 12, 22, 64, 11] [25, 12, 22, 64, 11] swap 64 > 11 → [25, 12, 22, 11, 64] Pass 2: [25, 12, 22, 11, 64] swap → [12, 25, 22, 11, 64] [12, 25, 22, 11, 64] swap → [12, 22, 25, 11, 64] [12, 22, 25, 11, 64] swap → [12, 22, 11, 25, 64] Pass 3: [12, 22, 11, 25, 64] → [12, 11, 22, 25, 64] Pass 4: [12, 11, 22, 25, 64] → [11, 12, 22, 25, 64] ✔ Sorted!
Key Insight
The early termination optimization is crucial: if no swaps occur during an entire pass, the array is already sorted. This gives Bubble Sort its best-case O(n) time on already-sorted input.

Implementation

BubbleSelectionInsertion.java
static void bubbleSort(int[] arr) { int n = arr.length; for (int i = 0; i < n - 1; i++) { boolean swapped = false; for (int j = 0; j < n - 1 - i; j++) { if (arr[j] > arr[j + 1]) { int temp = arr[j]; arr[j] = arr[j + 1]; arr[j + 1] = temp; swapped = true; } } if (!swapped) break; // already sorted -- early termination } }
sorting.cpp
void bubbleSort(vector<int>& arr) { int n = arr.size(); for (int i = 0; i < n - 1; ++i) { bool swapped = false; for (int j = 0; j < n - 1 - i; ++j) { if (arr[j] > arr[j + 1]) { swap(arr[j], arr[j + 1]); swapped = true; } } if (!swapped) break; // early termination if already sorted } }
sorting.py
def bubble_sort(arr: List[int]) -> List[int]: n = len(arr) for i in range(n): swapped = False for j in range(0, n - 1 - i): if arr[j] > arr[j + 1]: arr[j], arr[j + 1] = arr[j + 1], arr[j] swapped = True if not swapped: break # early termination return arr
sorting.rs
fn bubble_sort(arr: &mut Vec<i32>) { let n = arr.len(); for i in 0..n { let mut swapped = false; for j in 0..n.saturating_sub(i + 1) { if arr[j] > arr[j + 1] { arr.swap(j, j + 1); swapped = true; } } if !swapped { break; // Array is sorted } } }

🔍 Selection Sort

Selection Sort divides the array into sorted (left) and unsorted (right) portions. On each pass, it finds the minimum element in the unsorted portion and swaps it into the correct position at the end of the sorted portion.

Real-World Analogy

Like picking cards from a messy pile: scan all the cards, find the smallest one, place it first. Then scan the remaining cards, find the next smallest, place it second. Repeat.

Step-by-Step Visualization

Array: [64, 25, 12, 22, 11] Pass 1: Find min in [64,25,12,22,11] → 11 at index 4 Swap arr[0] and arr[4]: [11, 25, 12, 22, 64] Pass 2: Find min in [25,12,22,64] → 12 at index 2 Swap arr[1] and arr[2]: [11, 12, 25, 22, 64] Pass 3: Find min in [25,22,64] → 22 at index 3 Swap arr[2] and arr[3]: [11, 12, 22, 25, 64] Pass 4: Find min in [25,64] → 25 already in place [11, 12, 22, 25, 64] ✔ Sorted!
Key Insight
Selection Sort performs at most O(n) swaps, which makes it optimal when write operations are expensive (e.g., writing to flash memory). However, it always does O(n^2) comparisons regardless of input order. It is also not stable in its basic form.

Implementation

BubbleSelectionInsertion.java
static void selectionSort(int[] arr) { int n = arr.length; for (int i = 0; i < n - 1; i++) { int minIdx = i; for (int j = i + 1; j < n; j++) { if (arr[j] < arr[minIdx]) minIdx = j; } if (minIdx != i) { int temp = arr[i]; arr[i] = arr[minIdx]; arr[minIdx] = temp; } } }
sorting.cpp
void selectionSort(vector<int>& arr) { int n = arr.size(); for (int i = 0; i < n - 1; ++i) { int minIdx = i; for (int j = i + 1; j < n; ++j) { if (arr[j] < arr[minIdx]) minIdx = j; } if (minIdx != i) swap(arr[i], arr[minIdx]); } }
sorting.py
def selection_sort(arr: List[int]) -> List[int]: n = len(arr) for i in range(n): min_idx = i for j in range(i + 1, n): if arr[j] < arr[min_idx]: min_idx = j arr[i], arr[min_idx] = arr[min_idx], arr[i] return arr
sorting.rs
fn selection_sort(arr: &mut Vec<i32>) { let n = arr.len(); for i in 0..n { let min_idx = (i..n).min_by_key(|&j| arr[j]).unwrap(); arr.swap(i, min_idx); } }

📋 Insertion Sort

Insertion Sort builds the sorted array one element at a time. It picks the next unsorted element and inserts it into its correct position within the already-sorted left portion, shifting elements as needed.

Real-World Analogy

Exactly like sorting a hand of playing cards: you pick up one card at a time and slide it into the correct position among the cards you have already sorted in your hand.

Step-by-Step Visualization

Array: [64, 25, 12, 22, 11] Step 1: key = 25, insert into [64] 25 < 64, shift 64 right → [25, 64, 12, 22, 11] Step 2: key = 12, insert into [25, 64] 12 < 64, shift; 12 < 25, shift → [12, 25, 64, 22, 11] Step 3: key = 22, insert into [12, 25, 64] 22 < 64, shift; 22 < 25, shift; 22 > 12, stop → [12, 22, 25, 64, 11] Step 4: key = 11, insert into [12, 22, 25, 64] Shift all → [11, 12, 22, 25, 64] ✔ Sorted!
Key Insight
Insertion Sort is the best choice for small arrays (n < 16) and nearly-sorted data. Java's Arrays.sort() for objects uses TimSort, which switches to Insertion Sort for small subarrays. The number of swaps equals the number of inversions in the array.

Implementation

BubbleSelectionInsertion.java
static void insertionSort(int[] arr) { int n = arr.length; for (int i = 1; i < n; i++) { int key = arr[i]; // element to insert int j = i - 1; // Shift elements greater than key one position right while (j >= 0 && arr[j] > key) { arr[j + 1] = arr[j]; j--; } arr[j + 1] = key; // insert key at correct position } }
sorting.cpp
void insertionSort(vector<int>& arr) { int n = arr.size(); for (int i = 1; i < n; ++i) { int key = arr[i]; int j = i - 1; while (j >= 0 && arr[j] > key) { arr[j + 1] = arr[j]; --j; } arr[j + 1] = key; } }
sorting.py
def insertion_sort(arr: List[int]) -> List[int]: for i in range(1, len(arr)): key = arr[i] j = i - 1 while j >= 0 and arr[j] > key: arr[j + 1] = arr[j] j -= 1 arr[j + 1] = key return arr
sorting.rs
fn insertion_sort(arr: &mut Vec<i32>) { for i in 1..arr.len() { let key = arr[i]; let mut j = i; while j > 0 && arr[j - 1] > key { arr[j] = arr[j - 1]; j -= 1; } arr[j] = key; } }

✍️ Merge Sort

Merge Sort is a divide-and-conquer algorithm. It splits the array in half recursively until each piece has one element (trivially sorted), then merges sorted halves back together.

Algorithm

  1. DIVIDE: Split the array in half recursively until each subarray has size 1.
  2. CONQUER: Merge two sorted halves into one sorted array using two pointers.

Step-by-Step Visualization

[38, 27, 43, 3, 9, 82, 10] / \ [38, 27, 43, 3] [9, 82, 10] / \ / \ [38, 27] [43, 3] [9, 82] [10] / \ / \ / \ | [38] [27] [43] [3] [9] [82] [10] \ / \ / \ / | [27, 38] [3, 43] [9, 82] [10] \ / \ / [3, 27, 38, 43] [9, 10, 82] \ / [3, 9, 10, 27, 38, 43, 82] ✔

Recurrence Relation

T(n) = 2T(n/2) + O(n) -- By the Master Theorem, this solves to O(n log n).

Key Insight
Merge Sort guarantees O(n log n) in all cases (best, average, worst). It is stable (equal elements maintain their relative order) and is the basis for TimSort, used by Java's Arrays.sort() for objects and Python's built-in sort(). The tradeoff is O(n) extra space for the temporary arrays.

Implementation

MergeSort.java
static void merge(int[] arr, int left, int mid, int right) { int n1 = mid - left + 1; int n2 = right - mid; int[] L = Arrays.copyOfRange(arr, left, mid + 1); int[] R = Arrays.copyOfRange(arr, mid + 1, right + 1); int i = 0, j = 0, k = left; while (i < n1 && j < n2) { if (L[i] <= R[j]) arr[k++] = L[i++]; // <= keeps it stable else arr[k++] = R[j++]; } while (i < n1) arr[k++] = L[i++]; while (j < n2) arr[k++] = R[j++]; } static void mergeSort(int[] arr, int left, int right) { if (left >= right) return; // base case: 0 or 1 element int mid = left + (right - left) / 2; mergeSort(arr, left, mid); mergeSort(arr, mid + 1, right); merge(arr, left, mid, right); }
sorting.cpp
long long mergeAndCount(vector<int>& arr, int left, int mid, int right) { vector<int> temp; temp.reserve(right - left + 1); int i = left, j = mid + 1; long long inversions = 0; while (i <= mid && j <= right) { if (arr[i] <= arr[j]) { temp.push_back(arr[i++]); } else { inversions += (mid - i + 1); temp.push_back(arr[j++]); } } while (i <= mid) temp.push_back(arr[i++]); while (j <= right) temp.push_back(arr[j++]); copy(temp.begin(), temp.end(), arr.begin() + left); return inversions; } long long mergeSortAndCount(vector<int>& arr, int left, int right) { if (left >= right) return 0; int mid = left + (right - left) / 2; long long inversions = 0; inversions += mergeSortAndCount(arr, left, mid); inversions += mergeSortAndCount(arr, mid + 1, right); inversions += mergeAndCount(arr, left, mid, right); return inversions; }
sorting.py
def merge_sort(arr: List[int]) -> List[int]: if len(arr) <= 1: return arr mid = len(arr) // 2 left = arr[:mid] right = arr[mid:] merge_sort(left) merge_sort(right) # Merge back into arr i = j = k = 0 while i < len(left) and j < len(right): if left[i] <= right[j]: arr[k] = left[i] i += 1 else: arr[k] = right[j] j += 1 k += 1 while i < len(left): arr[k] = left[i]; i += 1; k += 1 while j < len(right): arr[k] = right[j]; j += 1; k += 1 return arr
sorting.rs
fn merge_sort(arr: &[i32]) -> (Vec<i32>, u64) { if arr.len() <= 1 { return (arr.to_vec(), 0); } let mid = arr.len() / 2; let (left_sorted, left_inv) = merge_sort(&arr[..mid]); let (right_sorted, right_inv) = merge_sort(&arr[mid..]); let (merged, split_inv) = merge(&left_sorted, &right_sorted); (merged, left_inv + right_inv + split_inv) } fn merge(left: &[i32], right: &[i32]) -> (Vec<i32>, u64) { let mut result = Vec::with_capacity(left.len() + right.len()); let mut inversions: u64 = 0; let (mut i, mut j) = (0, 0); while i < left.len() && j < right.len() { if left[i] <= right[j] { result.push(left[i]); i += 1; } else { result.push(right[j]); inversions += (left.len() - i) as u64; j += 1; } } result.extend_from_slice(&left[i..]); result.extend_from_slice(&right[j..]); (result, inversions) }

⚡ Quick Sort

Quick Sort is another divide-and-conquer algorithm. It selects a pivot element, partitions the array so elements less than the pivot go left and greater go right, then recursively sorts each partition.

Partition Schemes

Lomuto Partition (simpler, uses last element as pivot)

Array: [10, 7, 8, 9, 1, 5] pivot = 5 (last element) i = -1 (boundary of "less than pivot" region) j=0: arr[0]=10 > 5, skip j=1: arr[1]=7 > 5, skip j=2: arr[2]=8 > 5, skip j=3: arr[3]=9 > 5, skip j=4: arr[4]=1 <= 5, i++ then swap arr[0]↔arr[4] → [1, 7, 8, 9, 10, 5] Place pivot: swap arr[i+1]↔arr[5] → [1, 5, 8, 9, 10, 7] Pivot 5 is now at index 1 (its final position!)

Hoare Partition (original, fewer swaps, uses first element as pivot)

Uses two pointers converging from both ends. On average, Hoare performs about 3x fewer swaps than Lomuto.

Key Insight
Randomized Quick Sort avoids the O(n^2) worst case by randomly selecting the pivot before partitioning. This makes the expected time O(n log n) regardless of input distribution. Java uses a Dual-Pivot Quicksort (Yaroslavskiy) for primitive arrays.
Common Pitfall
Quick Sort degrades to O(n^2) when the pivot is always the smallest or largest element -- which happens on already-sorted or reverse-sorted input if you always pick the first or last element as pivot. Always use randomized pivot selection in practice.

Implementation

QuickSort.java
static int lomutoPartition(int[] arr, int low, int high) { int pivot = arr[high]; int i = low - 1; for (int j = low; j < high; j++) { if (arr[j] <= pivot) { i++; int temp = arr[i]; arr[i] = arr[j]; arr[j] = temp; } } int temp = arr[i + 1]; arr[i + 1] = arr[high]; arr[high] = temp; return i + 1; } // Randomized Quicksort -- avoids O(n^2) worst case static void quickSortRandom(int[] arr, int low, int high) { if (low >= high) return; int pivotIdx = low + rand.nextInt(high - low + 1); int temp = arr[pivotIdx]; arr[pivotIdx] = arr[high]; arr[high] = temp; int p = lomutoPartition(arr, low, high); quickSortRandom(arr, low, p - 1); quickSortRandom(arr, p + 1, high); }
sorting.cpp
int lomutoPartition(vector<int>& arr, int lo, int hi) { int pivot = arr[hi]; int i = lo - 1; for (int j = lo; j < hi; ++j) { if (arr[j] <= pivot) { swap(arr[++i], arr[j]); } } swap(arr[i + 1], arr[hi]); return i + 1; } int randomizedPartition(vector<int>& arr, int lo, int hi) { static mt19937 rng(42); int randIdx = uniform_int_distribution<int>(lo, hi)(rng); swap(arr[randIdx], arr[hi]); return lomutoPartition(arr, lo, hi); } void quickSortHelper(vector<int>& arr, int lo, int hi) { if (lo >= hi) return; int pivot = randomizedPartition(arr, lo, hi); quickSortHelper(arr, lo, pivot - 1); quickSortHelper(arr, pivot + 1, hi); }
sorting.py

sorting.rs
fn lomuto_partition(arr: &mut [i32]) -> usize { let n = arr.len(); let pivot = arr[n - 1]; let mut i = 0; for j in 0..n - 1 { if arr[j] < pivot { arr.swap(i, j); i += 1; } } arr.swap(i, n - 1); i } fn quicksort(arr: &mut [i32]) { if arr.len() <= 1 { return; } let pivot_idx = lomuto_partition(arr); quicksort(&mut arr[..pivot_idx]); if pivot_idx + 1 < arr.len() { quicksort(&mut arr[pivot_idx + 1..]); } }

🏆 QuickSelect -- Kth Smallest Element

QuickSelect uses the Quick Sort partitioning idea to find the kth smallest element in O(n) average time without fully sorting the array. After partitioning, the pivot is at its final sorted position. If that position equals k, we are done. Otherwise, recurse on the correct side only.

QuickSort.java
static int quickSelect(int[] arr, int low, int high, int k) { if (low == high) return arr[low]; int pivotIdx = low + rand.nextInt(high - low + 1); int temp = arr[pivotIdx]; arr[pivotIdx] = arr[high]; arr[high] = temp; int p = lomutoPartition(arr, low, high); int rank = p - low + 1; if (rank == k) return arr[p]; else if (k < rank) return quickSelect(arr, low, p - 1, k); else return quickSelect(arr, p + 1, high, k - rank); }
sorting.cpp
int quickSelect(vector<int>& arr, int lo, int hi, int k) { while (lo <= hi) { int pivotIdx = randomizedPartition(arr, lo, hi); if (pivotIdx == k) return arr[pivotIdx]; else if (pivotIdx < k) lo = pivotIdx + 1; else hi = pivotIdx - 1; } return -1; }
sorting.py
def quickselect(arr: List[int], k: int) -> int: target = k - 1 # convert to 0-indexed lo, hi = 0, len(arr) - 1 while lo <= hi: rand_idx = random.randint(lo, hi) arr[rand_idx], arr[hi] = arr[hi], arr[rand_idx] pivot_idx = _lomuto_partition(arr, lo, hi) if pivot_idx == target: return arr[pivot_idx] elif pivot_idx < target: lo = pivot_idx + 1 else: hi = pivot_idx - 1 return arr[lo]
sorting.rs
fn quickselect(arr: &mut [i32], k: usize) -> i32 { assert!(k < arr.len(), "k must be within array bounds"); if arr.len() == 1 { return arr[0]; } let pivot_idx = lomuto_partition(arr); match k.cmp(&pivot_idx) { std::cmp::Ordering::Equal => arr[pivot_idx], std::cmp::Ordering::Less => quickselect(&mut arr[..pivot_idx], k), std::cmp::Ordering::Greater => quickselect(&mut arr[pivot_idx + 1..], k - pivot_idx - 1), } }

🔢 Counting Sort

Counting Sort is a non-comparison-based sorting algorithm. Instead of comparing elements, it counts the occurrence of each value, then reconstructs the sorted array. It runs in O(n + k) time where k is the range of input values.

Key Insight
Counting Sort breaks the O(n log n) lower bound of comparison-based sorts. It is extremely fast when the range k is small relative to n. It is stable, which makes it the building block for Radix Sort.

Implementation

sorting.cpp
vector<int> countingSort(const vector<int>& arr) { if (arr.empty()) return {}; auto [minIt, maxIt] = minmax_element(arr.begin(), arr.end()); int minVal = *minIt, maxVal = *maxIt; int range = maxVal - minVal + 1; vector<int> count(range, 0); for (int x : arr) count[x - minVal]++; // Build prefix sum for stable sort for (int i = 1; i < range; ++i) count[i] += count[i - 1]; // Build output in reverse for stability vector<int> output(arr.size()); for (int i = (int)arr.size() - 1; i >= 0; --i) { output[--count[arr[i] - minVal]] = arr[i]; } return output; }
sorting.py
def counting_sort(arr: List[int]) -> List[int]: if not arr: return arr max_val = max(arr) count = [0] * (max_val + 1) for val in arr: count[val] += 1 for i in range(1, len(count)): count[i] += count[i - 1] output = [0] * len(arr) for val in reversed(arr): count[val] -= 1 output[count[val]] = val arr[:] = output return arr
sorting.rs
fn counting_sort(arr: &mut Vec<i32>) { if arr.is_empty() { return; } let &max_val = arr.iter().max().unwrap(); let &min_val = arr.iter().min().unwrap(); let range = (max_val - min_val + 1) as usize; let mut count = vec![0usize; range]; for &val in arr.iter() { count[(val - min_val) as usize] += 1; } let mut idx = 0; for i in 0..range { for _ in 0..count[i] { arr[idx] = i as i32 + min_val; idx += 1; } } }

Interview Tip
When asked "which sort to use?", consider: (1) If data is nearly sorted, Insertion Sort is O(n). (2) If stability matters, Merge Sort. (3) For general-purpose in-memory sorting, Quick Sort (randomized) -- it has better cache performance. (4) For integer data with small range, Counting Sort.

🔄 Counting Inversions

An inversion is a pair (i, j) where i < j but arr[i] > arr[j]. The number of inversions measures how "unsorted" an array is. A sorted array has 0 inversions; a reverse-sorted array of n elements has n*(n-1)/2 inversions.

Efficient Counting with Merge Sort

During the merge step, whenever we pick from the right half before the left half, every remaining element in the left half forms an inversion with that right element. This counts all split inversions in O(n log n) total.

Inversions in [2, 4, 1, 3, 5]: (2,1), (4,1), (4,3) = 3 inversions Inversions in [5, 4, 3, 2, 1]: Every pair is an inversion = 5*4/2 = 10 inversions Inversions in [1, 2, 3]: Already sorted = 0 inversions
