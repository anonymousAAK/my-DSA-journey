Sliding Window Pattern

WEEK 30 - WEB (JS) ADVANCED TOPICS

CONCEPT: Maintain a window [l, r] over a sequence; grow on the right, shrink on the left when an invariant is violated. Each element enters/leaves at most once -> O(n).

KEY POINTS: Map for counts; deque for monotonic structures; canonical LeetCode problems.

ALGORITHM: for r in 0..n: add a[r]; while invariant violated: remove a[l]; l++.

JS-SPECIFIC NOTES: Map is faster than {} for character counts since strings hash predictably; Array as deque using push/shift is O(n) shift but ok for small windows.

DRY RUN: "abcabcbb" -> 3; minWindow -> "BANC"; maxSlidingWindow -> [3,3,5,5,6,7].

COMPLEXITY: Time O(n).
