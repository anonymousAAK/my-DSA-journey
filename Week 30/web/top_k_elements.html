Top-K Elements Pattern

WEEK 30 - WEB (JS) ADVANCED TOPICS

CONCEPT: Find K largest / most-frequent elements via min-heap (O(n log K)), bucket sort by frequency (O(n)), or quickselect (O(n) expected).

KEY POINTS: Min-heap of size K keeps largest; buckets of frequency are bounded by n; quickselect partitions in place.

ALGORITHM: see code for findKthLargest, topKFrequent (heap and bucket), quickselectKthLargest.

JS-SPECIFIC NOTES: JS lacks a built-in heap; we ship a small MinHeap class. For top-K frequent, Map gives O(1) tally.

DRY RUN: findKthLargest [3,2,1,5,6,4],k=2 -> 5. topKFrequent [1,1,1,2,2,3],k=2 -> [1, 2] (any order).

COMPLEXITY: Heap O(n log K); Bucket O(n); Quickselect O(n) expected.
