Heap Fundamentals

A heap is a complete binary tree stored as an array where every parent satisfies the heap property relative to its children:

  • Max-Heap: parent ≥ children (root = maximum element)
  • Min-Heap: parent ≤ children (root = minimum element)

Array Representation (0-indexed)

Since a heap is a complete binary tree, it can be stored compactly in an array with no gaps:

  • Parent of i: (i - 1) / 2
  • Left child of i: 2 * i + 1
  • Right child of i: 2 * i + 2

Key Insight — Why Arrays?
A complete binary tree has no gaps, so storing it in an array wastes no space and provides O(1) access to any node's parent or children via simple arithmetic. No pointers needed!

Core Operations

  • Sift Up (after insert): New element placed at end, bubbled up by swapping with parent while larger
  • Sift Down (after extract): Root replaced by last element, pushed down by swapping with larger child
  • Build Heap (Floyd's algorithm): Start from last non-leaf, sift down each. O(n) — not O(n log n)!

Key Insight — Build Heap is O(n), not O(n log n)

Floyd's build-heap algorithm sifts down from the last non-leaf to the root. Most nodes are near the bottom and sift down only a short distance. The total work sums to O(n) by a geometric series argument. This is important for heap sort's efficiency.

Kth Largest Element


Find the kth largest element in an unsorted array. Strategy: maintain a min-heap of size k. After processing all elements, the root is the kth largest.



Algorithm


    

  1. Push each element onto a min-heap
    2. 

  2. If heap size exceeds k, pop the minimum (smallest of the k largest)
    3. 

  3. After all elements, the root = kth largest
    4. 



Time: O(n log k), Space: O(k)





Key Insight — Min-Heap for Kth Largest

It seems counterintuitive to use a min-heap to find the kth largest. The trick: the min-heap holds the k largest elements seen so far. Its root (the minimum of those k elements) is exactly the kth largest overall. By evicting the smallest of the top-k candidates at each step, we guarantee correctness.

Key Insight — Python's heapq is a min-heap

Python only has a min-heap (heapq). To simulate a max-heap, negate all values before pushing and negate again when popping. This is the standard Python trick: heappush(max_heap, -val) and -heappop(max_heap).

Week 15 Cheat Sheet




Heap Property
Max-heap: parent ≥ children. Min-heap: parent ≤ children. Stored as array.


Index Formulas
Parent: (i-1)/2. Left: 2i+1. Right: 2i+2.


Heap Sort
Build heap O(n) + extract n times O(n log n). In-place. Not stable.


Kth Largest
Min-heap of size k. Root = kth largest. O(n log k).


Median Finder
Max-heap (lower) + min-heap (upper). Add O(log n), find O(1).


Language Tips
Java: PriorityQueue. Python: heapq (min). C++: priority_queue (max). Rust: BinaryHeap (max).
