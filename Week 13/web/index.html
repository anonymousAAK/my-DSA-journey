Queue Fundamentals

A queue is a linear data structure that follows the FIFO (First In, First Out) principle. Think of a line at a grocery store: the first person to join the line is the first to be served.

Core Operations (All O(1))

  • enqueue(x) / offer(x) — Add an element to the rear
  • dequeue() / poll() — Remove and return the front element
  • peek() / front() — View the front element without removing
  • isEmpty() — Check if the queue is empty

Queue Variants

  • Circular Queue — Array-based queue that wraps around using modular arithmetic
  • Deque (Double-Ended Queue) — Insert/delete at BOTH ends
  • Priority Queue — Elements dequeued by priority (covered in Week 15)

Common Applications

  • BFS (Breadth-First Search)
  • Level-order tree traversal
  • Task scheduling / job queues
  • Sliding window problems
Key Insight
In Java, use ArrayDeque as your Queue implementation (not LinkedList). In Python, use collections.deque. In Rust, use VecDeque. In C++, use std::queue or std::deque. All provide O(1) amortized enqueue/dequeue. 
















Circular Queue (Ring Buffer)


A regular array-based queue wastes space as elements are dequeued from the front. A circular queue solves this by wrapping the indices around using modular arithmetic: index = (index + 1) % capacity.



Why Circular?


In a naive array-based queue, after dequeuing many elements, the front of the array is wasted space. You would need to shift all elements left (O(n)) or eventually run out of space. A circular queue reuses freed positions by letting the rear pointer wrap around to the beginning of the array.

Key Insight

The wrap-around formula rear = (rear + 1) % capacity is the heart of the circular queue. When the rear index reaches the end of the array, it wraps back to index 0, reusing freed space. The same formula applies to the front pointer on dequeue.


























Edge Cases to Watch


    

  • Queue full: size == capacity. Must check before enqueue.
    • 

  • Queue empty: size == 0. Must check before dequeue/peek.
    • 

  • Wrap-around: After the rear pointer passes the last index, it wraps to 0.
    • 

  • Single element: Front and rear point to the same slot.
    • 









Queue Using Two Stacks


This is a classic interview problem: implement a FIFO queue using only LIFO stacks. The trick is to use two stacks — an inbox for enqueue and an outbox for dequeue.



How It Works


    

  1. enqueue(x): Push onto the inbox stack. O(1).
    2. 

  2. dequeue(): If the outbox is empty, pour ALL elements from inbox to outbox (this reverses the order, giving FIFO behavior). Pop from outbox.
    3. 

  3. Each element is moved at most twice (once into inbox, once into outbox), so the amortized cost per operation is O(1).
    4.

Key Insight — Amortized O(1)

Even though a single dequeue can take O(n) when the transfer happens, each element is moved from inbox to outbox exactly once. Over a sequence of n operations, the total work is O(n), giving an amortized cost of O(1) per operation. This is similar to how dynamic array resizing gives amortized O(1) append.































Sliding Window Maximum (Monotonic Deque)


Given an array of n integers and a window size k, find the maximum in each window of size k as it slides across the array. This is a classic hard problem that can be solved optimally with a monotonic deque.



Naive Approach: O(n * k)


For each window position, scan all k elements to find the max. Too slow for large inputs.



Optimal Approach: O(n) with Monotonic Deque


Maintain a deque of indices where the corresponding values are in decreasing order. The front of the deque always holds the index of the maximum in the current window.



Algorithm


    

  1. For each element arr[i]:
    2. 

  2. Remove expired: Remove indices from the front that are outside the current window
    3. 

  3. Maintain monotonicity: Remove indices from the back whose values are smaller than arr[i] (they can never be the window maximum while arr[i] is in the window)
    4. 

  4. Add current: Push index i to the back
    5. 

  5. Record result: Once i >= k-1, the front of the deque is the window maximum
    6.

Key Insight — Why O(n)?

Each element is added to the deque at most once and removed at most once. So across all iterations, the total number of deque operations is at most 2n, giving O(n) overall. The deque stores at most k indices at any time, so space is O(k).


























Common Mistakes


    

  • Storing values instead of indices in the deque — you need indices to check if elements are inside the window
    • 

  • Forgetting to remove expired indices from the front of the deque
    • 

  • Using < instead of <= when removing smaller elements (both work but affect how duplicates are handled)

Week 13 Cheat Sheet






Queue Basics


FIFO: first in, first out. enqueue at rear, dequeue from front. All O(1).






Circular Queue


Fixed-size ring buffer. rear = (rear+1) % cap. Reuses freed space.






Two-Stack Queue


Inbox + outbox. Transfer reverses order for FIFO. Amortized O(1).






Monotonic Deque


Decreasing deque of indices. Front = max. Each element enters/exits once → O(n).






Language Tips


Java: ArrayDeque. Python: collections.deque. C++: std::deque. Rust: VecDeque.






When to Use


BFS, level-order traversal, task scheduling, sliding window problems.
