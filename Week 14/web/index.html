Tree Fundamentals

A tree is a hierarchical data structure consisting of nodes connected by edges. A binary tree is a tree where each node has at most 2 children: left and right.

Terminology

  • Root — Top node with no parent
  • Leaf — Node with no children
  • Height — Longest path from root to any leaf
  • Depth — Distance from root to a node
  • Level = Depth + 1

Key Insight — Recursive Template

All three DFS traversals share the same recursive structure. The only difference is when you process the current node: preorder = before children, inorder = between children, postorder = after children. For iterative versions, use an explicit stack.






























Tree Properties: Height, Diameter, Mirror



Height


Height = number of nodes on the longest root-to-leaf path. Empty tree has height 0. Computed recursively: height(node) = 1 + max(height(left), height(right)). Time: O(n), Space: O(h).



Diameter


The diameter is the longest path between any two nodes (measured in edges). This path may or may not pass through the root. At each node, the diameter through that node is leftHeight + rightHeight. Track the global maximum.

Mirror / Invert


Swap left and right children at every node. A simple recursive swap. This is the famous "invert a binary tree" interview question.





























Binary Search Trees (BST)


A BST is a binary tree where for every node: all values in the left subtree are smaller, and all values in the right subtree are larger. This property enables O(h) search, insert, and delete.

Key Insight

The BST property means inorder traversal always produces sorted output. This is the key to many BST algorithms: validate BST, find kth smallest, find successor/predecessor.









BST Operations: Insert, Search, Delete



Insert — O(h)


Search for the correct leaf position and add the new node. Compare with each node: go left if smaller, right if larger.



Search — O(h)


Compare target with current node. If equal, found. If smaller, search left. If larger, search right.



Delete — O(h)


Three cases:


    

  1. Leaf node: Simply remove it
    2. 

  2. One child: Replace with the child
    3. 

  3. Two children: Replace with the inorder successor (smallest node in right subtree), then delete the successor
    4. 






























Validate BST & Lowest Common Ancestor



Validate BST


A tree is a valid BST if every node's value falls within a valid range. Pass min/max bounds down: the left child must be less than the current node, the right child must be greater.



LCA in BST — O(h)


The Lowest Common Ancestor of nodes p and q in a BST:


    

  • If both p, q < root: LCA is in the left subtree
    • 

  • If both p, q > root: LCA is in the right subtree
    • 

  • Otherwise: root is the LCA (the split point)
    • 




Kth Smallest in BST


Use inorder traversal (which gives sorted order) and stop at the kth element. Time: O(h + k).

Week 14 Cheat Sheet




Traversals
Inorder: L-Root-R (sorted for BST). Preorder: Root-L-R. Postorder: L-R-Root. Level: BFS with queue.


BST Property
Left subtree < node < right subtree. Enables O(h) search, insert, delete.


Height & Diameter
Height: 1 + max(L, R). Diameter: max of leftH + rightH at each node.


BST Delete
Leaf: remove. One child: replace. Two children: replace with inorder successor.


Validate BST
Pass min/max bounds or check inorder is strictly increasing.


LCA in BST
Follow split point: both left → go left, both right → go right, else found. O(h).
