📝 String Basics

In Java, strings are immutable -- every modification creates a new object. Use StringBuilder for efficient repeated modifications.

Common Operations & Time Complexity

OperationTimeNotes
charAt(i)O(1)Direct index access
substring(l, r)O(r-l)Creates a new string
indexOf()O(n*m)Naive search; O(n) with KMP
concat (+)O(n)Creates new string each time!
StringBuilder.append()O(1) amortizedEfficient for loops
Golden Rule

Never concatenate strings inside a loop with +. Use StringBuilder instead. The difference is O(n) vs O(n^2) for building a string of length n.

🔄 Palindrome Check

A palindrome reads the same forwards and backwards: "racecar", "madam", "level".

The two-pointer technique is the optimal approach: one pointer from the start, one from the end, moving inward.

PalindromeAndAnagram.java
// Simple palindrome -- O(n) time, O(1) space static boolean isPalindrome(String s) { int left = 0, right = s.length() - 1; while (left < right) { if (s.charAt(left) != s.charAt(right)) return false; left++; right--; } return true; } // Ignoring non-alphanumeric characters and case // "A man, a plan, a canal: Panama" --> true static boolean isPalindromeIgnoreNonAlpha(String s) { int left = 0, right = s.length() - 1; while (left < right) { while (left < right && !Character.isLetterOrDigit(s.charAt(left))) left++; while (left < right && !Character.isLetterOrDigit(s.charAt(right))) right--; if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) return false; left++; right--; } return true; }
strings.cpp
// Alphanumeric palindrome -- O(n) time, O(1) space bool isPalindromeAlphanumeric(const string& s) { int left = 0, right = (int)s.size() - 1; while (left < right) { while (left < right && !isalnum(s[left])) ++left; while (left < right && !isalnum(s[right])) --right; if (tolower(s[left]) != tolower(s[right])) return false; ++left; --right; } return true; }

🔀 Anagram Check

Two strings are anagrams if one is a rearrangement of the other: "listen" and "silent", "eat" and "tea".

Three Approaches

ApproachTimeSpace
Sort both, compareO(n log n)O(n) or O(1)
Frequency count (26-char array)O(n)O(1)
HashMap (unicode support)O(n)O(k) unique chars
PalindromeAndAnagram.java
// Approach 1: Sort and compare -- O(n log n) static boolean isAnagramSort(String a, String b) { if (a.length() != b.length()) return false; char[] ca = a.toCharArray(), cb = b.toCharArray(); Arrays.sort(ca); Arrays.sort(cb); return Arrays.equals(ca, cb); } // Approach 2: Frequency count -- O(n), O(1) space static boolean isAnagramFreq(String a, String b) { if (a.length() != b.length()) return false; int[] freq = new int[26]; for (char c : a.toCharArray()) freq[c - 'a']++; for (char c : b.toCharArray()) { freq[c - 'a']--; if (freq[c - 'a'] < 0) return false; } return true; } // Approach 3: HashMap -- works for any characters static boolean isAnagramMap(String a, String b) { if (a.length() != b.length()) return false; Map<Character, Integer> map = new HashMap<>(); for (char c : a.toCharArray()) map.merge(c, 1, Integer::sum); for (char c : b.toCharArray()) { if (!map.containsKey(c) || map.get(c) == 0) return false; map.merge(c, -1, Integer::sum); } return true; }
strings.cpp
// Frequency array -- O(n) time, O(1) space bool isAnagramFreq(const string& a, const string& b) { if (a.size() != b.size()) return false; array<int, 26> freq{}; for (char c : a) freq[c - 'a']++; for (char c : b) freq[c - 'a']--; return all_of(freq.begin(), freq.end(), [](int f){ return f == 0; }); } // General (unicode) -- O(n) time, O(k) space bool isAnagramGeneral(const string& a, const string& b) { if (a.size() != b.size()) return false; unordered_map<char, int> freq; for (char c : a) freq[c]++; for (char c : b) { if (--freq[c] < 0) return false; } return true; }

use std::collections::HashMap; fn is_anagram_freq(a: &str, b: &str) -> bool { if a.len() != b.len() { return false; } let mut freq: HashMap<char, i32> = HashMap::new(); for ch in a.chars() { *freq.entry(ch).or_insert(0) += 1; } for ch in b.chars() { *freq.entry(ch).or_insert(0) -= 1; } freq.values().all(|&count| count == 0) }

🔍 KMP String Search Algorithm

Find all occurrences of a pattern P in text T. The naive approach is O(n*m); KMP (Knuth-Morris-Pratt) achieves O(n+m) by avoiding redundant comparisons.

Key Insight

When a mismatch occurs, we already know some characters of the text that we have matched. The LPS array tells us exactly where to resume, skipping re-examination of characters we already know match.

LPS Array (Failure Function)

LPS[i] = length of the longest proper prefix of pattern[0..i] that is also a suffix of pattern[0..i].

Pattern: A A A C A A A A Index: 0 1 2 3 4 5 6 7 LPS: 0 1 2 0 1 2 3 3 Example: i=2: "AAA" → prefix "AA" = suffix "AA" → lps[2] = 2 i=3: "AAAC" → no prefix matches suffix → lps[3] = 0 i=6: "AAACAAA" → prefix "AAA" = suffix "AAA" → lps[6] = 3
KMPSearch.java
// Build LPS array -- O(m) time, O(m) space static int[] buildLPS(String pattern) { int m = pattern.length(); int[] lps = new int[m]; int len = 0, i = 1; while (i < m) { if (pattern.charAt(i) == pattern.charAt(len)) { len++; lps[i] = len; i++; } else { if (len != 0) { len = lps[len - 1]; // try shorter prefix-suffix } else { lps[i] = 0; i++; } } } return lps; } // KMP search -- O(n + m) time, O(m) space static List<Integer> kmpSearch(String text, String pattern) { List<Integer> positions = new ArrayList<>(); int n = text.length(), m = pattern.length(); if (m == 0 || m > n) return positions; int[] lps = buildLPS(pattern); int i = 0, j = 0; while (i < n) { if (text.charAt(i) == pattern.charAt(j)) { i++; j++; } if (j == m) { positions.add(i - j); // found match! j = lps[j - 1]; // look for next occurrence } else if (i < n && text.charAt(i) != pattern.charAt(j)) { if (j != 0) { j = lps[j - 1]; // skip using LPS } else { i++; } } } return positions; }

// Build LPS array -- O(m) time vector<int> buildLPS(const string& pattern) { int m = pattern.size(); vector<int> lps(m, 0); int len = 0, i = 1; while (i < m) { if (pattern[i] == pattern[len]) { lps[i++] = ++len; } else { if (len != 0) len = lps[len - 1]; else lps[i++] = 0; } } return lps; } // KMP search -- O(n + m) time vector<int> kmpSearch(const string& text, const string& pattern) { vector<int> result; if (pattern.empty()) return result; int n = text.size(), m = pattern.size(); vector<int> lps = buildLPS(pattern); int i = 0, j = 0; while (i < n) { if (text[i] == pattern[j]) { ++i; ++j; } if (j == m) { result.push_back(i - j); j = lps[j - 1]; } else if (i < n && text[i] != pattern[j]) { if (j != 0) j = lps[j - 1]; else ++i; } } return result; }

fn build_lps(pattern: &[u8]) -> Vec<usize> { let m = pattern.len(); let mut lps = vec![0usize; m]; let mut length = 0; let mut i = 1; while i < m { if pattern[i] == pattern[length] { length += 1; lps[i] = length; i += 1; } else if length != 0 { length = lps[length - 1]; } else { lps[i] = 0; i += 1; } } lps } fn kmp_search(text: &str, pattern: &str) -> Vec<usize> { let mut matches = Vec::new(); if pattern.is_empty() { return matches; } let text = text.as_bytes(); let pattern = pattern.as_bytes(); let (n, m) = (text.len(), pattern.len()); if m > n { return matches; } let lps = build_lps(pattern); let (mut i, mut j) = (0, 0); while i < n { if text[i] == pattern[j] { i += 1; j += 1; } if j == m { matches.push(i - j); j = lps[j - 1]; } else if i < n && text[i] != pattern[j] { if j != 0 { j = lps[j - 1]; } else { i += 1; } } } matches }
Why KMP Matters

For a text of length 1,000,000 and a pattern of length 1,000, naive search may perform up to 1 billion comparisons. KMP guarantees at most ~1,001,000 comparisons. This matters in text editors, DNA sequence analysis, and log processing.

📚 Summary

  • String immutability in Java/Python means every modification creates a new object. Use StringBuilder/list for loops.
  • Two-pointer technique solves palindrome checking in O(n) time and O(1) space.
  • Frequency counting with a 26-element array is the optimal approach for anagram detection.
  • KMP algorithm achieves O(n+m) pattern matching by precomputing the LPS (failure function) array.
  • The LPS array captures the structure of the pattern itself -- how its prefixes relate to its suffixes.
  • String algorithms are foundational for text processing, bioinformatics, search engines, and compiler design.
