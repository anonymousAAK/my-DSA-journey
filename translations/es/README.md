# My DSA Journey — De principiante a investigación avanzada

> **30 semanas. 4 lenguajes. Tests de corrección reales, quizzes de mastery y un quickstart de 4 horas — aprende DSA como resolución de problemas, no como memorización.**

Una guía estructurada y completa de 30 semanas para dominar estructuras de datos y algoritmos — desde primeros principios hasta temas a nivel de investigación y mastery para entrevistas.

Todas las implementaciones están disponibles en **4 lenguajes**: Java, Python, C++ y Rust — con comentarios detallados, análisis de complejidad y casos de prueba.

---

## Empieza aquí

**¿Primera vez en este repo?** 30 semanas es un compromiso real. Antes de hacerlo, prueba una de estas dos rampas:

- **[QUICKSTART.md](QUICKSTART.md) — 4 horas, 8 problemas.** Una sesión enfocada que te lleva por toda la metodología (descomponer → reconocer un patrón → implementar → journal) sobre problemas representativos de cada nivel de dificultad. Si el bucle te funciona, el camino largo es solo más repeticiones del mismo.
- **[diagnostic.md](diagnostic.md) — 15 preguntas, ~25 minutos.** Un diagnóstico calibrado que te coloca en el currículo para no perder semanas reaprendiendo lo que ya sabes. Mapea tu puntaje a una semana inicial específica.

¿Ya convencido? Salta a [Learning Paths](#learning-paths) o ve directo a [Week 1](../../Week%201/).

**¿Buscas un doc específico?** Consulta [INDEX.md](../../INDEX.md) — el índice maestro de cada archivo del repo, agrupado por propósito.

## Cómo usar esta guía

1. **Principiantes**: empieza en la Semana 1 y avanza secuencialmente. No saltes semanas.
2. **Intermedios**: salta a la Semana 8 (Search) o Semana 11 (Listas) según tu nivel de comodidad.
3. **Avanzados**: las Semanas 18-24 cubren DP, grafos avanzados y temas a nivel de investigación.
4. **Prep de entrevistas**: sigue el Interview Prep path de arriba, luego haz la Semana 30.
5. Cada carpeta semanal está organizada en subcarpetas `java`, `python`, `cpp`, `rust` y `web`, con un archivo numerado por tema en cada lenguaje.
6. **Flujo de estudio recomendado por tema**: lee el archivo Java para el walkthrough canónico → lee el archivo Python/C++/Rust/HTML con el mismo número → ejecútalos todos y compara salidas → termina con el bloque `NOTES` al final de cada archivo para ver qué cambió idiomáticamente.
7. Cada archivo incluye: enunciado del problema → enfoque/pseudocódigo → implementación → análisis de complejidad.
8. **Aprendices de lenguajes**: compara implementaciones entre lenguajes para entender diferencias de diseño.
9. **Abre las páginas HTML**: cada semana tiene `web/index.html` con explicaciones interactivas, diagramas y código copiable.

10. Cada `Week N/README.md` ahora refleja el roadmap de temas de Java y define qué significa "paridad detallada" para Python/C++/Rust.
