# Weekly Competition — 2026-W21

This week's problem comes from the Week 13 (Queues) and Week 16 (Hash Tables
/ I/O) material: implementing the classic `tail` utility efficiently. The
trick is doing it in a single pass with bounded memory, which the obvious
"read everything into a list and slice the tail" solution fails at on a
20 GB log file.

## Problem statement

Implement a function with this signature (adapt to your language; for the
sample IO below we use Python notation):

```
tail(lines: int, file: str) -> List[str]
```

Return the last `lines` lines of the file `file`, in the same order they
appear in the file. Required properties:

- **One pass.** Each line is read from disk exactly once.
- **O(N) time** in the number of lines `N` in the file.
- **O(lines) auxiliary memory.** You may not keep the entire file in
  memory.
- Use a **circular buffer** (a fixed-size array with two indices, or
  equivalent). Solutions using a deque are accepted *if and only if* they
  document why the deque is being used and why it satisfies the memory
  bound.

Edge cases the grader checks:

- `lines = 0` → return an empty list.
- File has fewer than `lines` lines → return all lines.
- File ends without a trailing newline.
- Empty file → return an empty list.
- `lines` greater than `2^20`.

## Sample IO

### Example 1

File `app.log`:

```
[INFO] boot
[INFO] connect
[WARN] retry
[ERR ] timeout
[INFO] shutdown
```

```
tail(3, "app.log")
→ ["[WARN] retry", "[ERR ] timeout", "[INFO] shutdown"]
```

### Example 2

```
tail(10, "app.log")  # file has only 5 lines
→ ["[INFO] boot", "[INFO] connect", "[WARN] retry", "[ERR ] timeout", "[INFO] shutdown"]
```

### Example 3

```
tail(0, "app.log")
→ []
```
