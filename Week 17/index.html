🔮 Graph Representations

⚖ Graph Representation Comparison

Choosing the right representation is critical for performance. Here is a detailed comparison and both implementations in all four languages.

Detailed Comparison Table

When to Use Which

Adjacency Matrix: Use when the graph is dense (E is close to V²), when you need frequent O(1) edge existence checks, or for algorithms like Floyd-Warshall. Adjacency List: Use for sparse graphs (most real-world graphs), BFS/DFS traversals, and when memory is a concern.

Memory Warning

An adjacency matrix for V = 10,000 vertices requires 10,000 × 10,000 = 100 million entries. For sparse graphs (e.g., social networks with V = 1M), this is impractical. Always default to adjacency lists unless you have a specific reason to use a matrix.

🔍 BFS -- Breadth-First Search

BFS explores the graph level by level using a queue. It finds the shortest path in unweighted graphs.

Graph: 0 -- 1 -- 2 | | 3 -- 4 -- 5 BFS from 0: Level 0: [0] Level 1: [1, 3] Level 2: [2, 4] Level 3: [5] Distances: {0:0, 1:1, 3:1, 2:2, 4:2, 5:3}
GraphRepresentations.java
int[] shortestDistances(int src) { int[] dist = new int[V]; Arrays.fill(dist, -1); dist[src] = 0; Queue<Integer> queue = new LinkedList<>(); queue.offer(src); while (!queue.isEmpty()) { int u = queue.poll(); for (int v : adj.get(u)) { if (dist[v] == -1) { dist[v] = dist[u] + 1; queue.offer(v); } } } return dist; }

fn bfs(&self, start: usize) -> (Vec<usize>, Vec<i32>) { let mut visited = vec![false; self.num_vertices]; let mut distances = vec![-1i32; self.num_vertices]; let mut order = Vec::new(); let mut queue = VecDeque::new(); visited[start] = true; distances[start] = 0; queue.push_back(start); while let Some(u) = queue.pop_front() { order.push(u); for &v in &self.adj[u] { if !visited[v] { visited[v] = true; distances[v] = distances[u] + 1; queue.push_back(v); } } } (order, distances) }

🛣 Shortest Path in Unweighted Graph

BFS naturally finds the shortest path in an unweighted graph because it explores nodes level by level. The first time a node is reached, the path used is guaranteed to be the shortest. This section provides a complete BFS-based shortest path finder that also reconstructs the actual path.

Graph: 0 -- 1 -- 3 -- 5 | | 2 --------4 Shortest path from 0 to 5: BFS explores: 0 → {1, 2} → {3, 4} → {5} Path: 0 → 1 → 3 → 5 (length = 3) Alternative: 0 → 2 → 4 → 3 → 5 (length = 4, NOT shortest)
Path Reconstruction

To reconstruct the actual path (not just the distance), store a parent array during BFS. After BFS completes, trace back from the destination to the source through the parent pointers.

BFS Shortest Path with Path Reconstruction

ShortestPath.java
// Returns shortest path from src to dest, or empty list if unreachable List<Integer> shortestPath(int src, int dest) { int[] dist = new int[V]; int[] parent = new int[V]; Arrays.fill(dist, -1); Arrays.fill(parent, -1); dist[src] = 0; Queue<Integer> queue = new LinkedList<>(); queue.offer(src); while (!queue.isEmpty()) { int u = queue.poll(); if (u == dest) break; // early termination for (int v : adj.get(u)) { if (dist[v] == -1) { dist[v] = dist[u] + 1; parent[v] = u; queue.offer(v); } } } if (dist[dest] == -1) return Collections.emptyList(); // Reconstruct path by tracing parent pointers List<Integer> path = new ArrayList<>(); for (int at = dest; at != -1; at = parent[at]) path.add(at); Collections.reverse(path); return path; }
shortest_path.cpp
// Returns {distance, path} from src to dest std::pair<int, std::vector<int>> shortest_path(int src, int dest) const { std::vector<int> dist(n_, -1); std::vector<int> parent(n_, -1); dist[src] = 0; std::queue<int> q; q.push(src); while (!q.empty()) { int u = q.front(); q.pop(); if (u == dest) break; for (auto [v, w] : adj_[u]) { if (dist[v] == -1) { dist[v] = dist[u] + 1; parent[v] = u; q.push(v); } } } if (dist[dest] == -1) return {-1, {}}; std::vector<int> path; for (int at = dest; at != -1; at = parent[at]) path.push_back(at); std::reverse(path.begin(), path.end()); return {dist[dest], path}; }

/// Returns (distance, path) from src to dest. /// Returns (-1, vec![]) if dest is unreachable. fn shortest_path(&self, src: usize, dest: usize) -> (i32, Vec<usize>) { let mut dist = vec![-1i32; self.num_vertices]; let mut parent = vec![usize::MAX; self.num_vertices]; dist[src] = 0; let mut queue = VecDeque::new(); queue.push_back(src); while let Some(u) = queue.pop_front() { if u == dest { break; } for &v in &self.adj[u] { if dist[v] == -1 { dist[v] = dist[u] + 1; parent[v] = u; queue.push_back(v); } } } if dist[dest] == -1 { return (-1, vec![]); } let mut path = Vec::new(); let mut at = dest; while at != usize::MAX { path.push(at); at = parent[at]; } path.reverse(); (dist[dest], path) }
Important Limitation

BFS-based shortest path only works for unweighted (or unit-weight) graphs. For weighted graphs, you need Dijkstra's algorithm (non-negative weights) or Bellman-Ford (handles negative weights). These are covered in later weeks.

🔮 DFS -- Depth-First Search

DFS explores as deep as possible before backtracking. Uses recursion or an explicit stack. Time: O(V + E).

GraphRepresentations.java

🟢 Finding All Connected Components

Beyond just counting components, we often need to identify which vertices belong to each component. This is done by running DFS/BFS from each unvisited vertex and collecting all reachable nodes into a group.

🔄 Cycle Detection

Undirected Graph

During DFS, if we encounter a visited neighbor that is NOT the parent of the current node, there is a cycle (a back edge).

Directed Graph (3-state coloring)

Use 3 states: WHITE (unvisited), GRAY (in the current DFS stack), BLACK (fully processed). A back edge to a GRAY node indicates a cycle.

TopologicalSort.java -- Directed Cycle Detection
// States: 0=unvisited, 1=in-stack (being processed), 2=done boolean hasCycle() { int[] state = new int[V]; for (int i = 0; i < V; i++) if (state[i] == 0 && dfsDetectCycle(i, state)) return true; return false; } boolean dfsDetectCycle(int u, int[] state) { state[u] = 1; // mark as in-stack for (int v : adj.get(u)) { if (state[v] == 1) return true; // back edge -- cycle if (state[v] == 0 && dfsDetectCycle(v, state)) return true; } state[u] = 2; // fully processed return false; }

🔴 Cycle Detection in Directed Graphs -- DFS Coloring

Directed cycle detection requires the three-color (WHITE/GRAY/BLACK) approach because in directed graphs, visiting an already-visited node does not necessarily mean a cycle -- it could be a cross edge or forward edge. Only a back edge (to a GRAY node currently on the recursion stack) indicates a cycle.

The Three States: WHITE (0) = unvisited, not yet discovered GRAY (1) = in current DFS path (on recursion stack) BLACK (2) = fully processed, all descendants explored Edge Classification in DFS: Tree edge: WHITE neighbor → recurse into it Back edge: GRAY neighbor → CYCLE DETECTED! Forward edge: BLACK neighbor → safe (already processed) Cross edge: BLACK neighbor → safe (different branch)
Common Mistake

Do NOT use a simple visited boolean for directed graphs. In a directed graph, reaching an already-visited node via a different path is perfectly valid (cross edge). You need the three-color scheme to distinguish back edges from cross/forward edges.

Complete Directed Cycle Detection (All 4 Languages)

DirectedCycleDetection.java
// Detect cycle and optionally return the cycle path boolean hasCycleDirected() { int[] color = new int[V]; // 0=WHITE, 1=GRAY, 2=BLACK for (int i = 0; i < V; i++) { if (color[i] == 0) { if (dfsCycleDirected(i, color)) return true; } } return false; } boolean dfsCycleDirected(int u, int[] color) { color[u] = 1; // mark GRAY (entering recursion stack) for (int v : adj.get(u)) { if (color[v] == 1) { // Back edge to a node on the current path return true; } if (color[v] == 0 && dfsCycleDirected(v, color)) { return true; } // color[v] == 2 (BLACK): already fully processed, skip } color[u] = 2; // mark BLACK (leaving recursion stack) return false; } // Variant: find the actual cycle List<Integer> findCycle() { int[] color = new int[V]; int[] parent = new int[V]; Arrays.fill(parent, -1); for (int i = 0; i < V; i++) { if (color[i] == 0) { int cycleEnd = dfsFindCycle(i, color, parent); if (cycleEnd != -1) { List<Integer> cycle = new ArrayList<>(); cycle.add(cycleEnd); for (int at = parent[cycleEnd]; at != cycleEnd; at = parent[at]) cycle.add(at); cycle.add(cycleEnd); Collections.reverse(cycle); return cycle; } } } return Collections.emptyList(); } int dfsFindCycle(int u, int[] color, int[] parent) { color[u] = 1; for (int v : adj.get(u)) { if (color[v] == 1) { parent[v] = u; return v; } if (color[v] == 0) { parent[v] = u; int result = dfsFindCycle(v, color, parent); if (result != -1) return result; } } color[u] = 2; return -1; }

Cycle Detection vs Topological Sort

Cycle detection and topological sort are closely related. A DAG (Directed Acyclic Graph) has a valid topological ordering if and only if it has no cycles. Kahn's algorithm naturally detects cycles: if the result contains fewer than V vertices, a cycle exists.

🎨 Bipartite Check

A graph is bipartite (2-colorable) if vertices can be split into two sets such that no edge connects vertices in the same set. Equivalent to having no odd-length cycles.

Key Insight
Use BFS with alternating colors. If any neighbor has the same color as the current node, the graph is NOT bipartite.
GraphRepresentations.java
boolean isBipartite() { int[] color = new int[V]; // 0=uncolored, 1=red, -1=blue for (int i = 0; i < V; i++) { if (color[i] == 0) { Queue<Integer> queue = new LinkedList<>(); queue.offer(i); color[i] = 1; while (!queue.isEmpty()) { int u = queue.poll(); for (int v : adj.get(u)) { if (color[v] == 0) { color[v] = -color[u]; queue.offer(v); } else if (color[v] == color[u]) return false; } } } } return true; }

fn is_bipartite(&self) -> bool { let mut color = vec![-1i32; self.num_vertices]; for start in 0..self.num_vertices { if color[start] != -1 { continue; } let mut queue = VecDeque::new(); color[start] = 0; queue.push_back(start); while let Some(u) = queue.pop_front() { for &v in &self.adj[u] { match color[v] { -1 => { color[v] = 1 - color[u]; queue.push_back(v); } c if c == color[u] => { return false; } _ => {} } } } } true }

🌈 Bipartite Graph Check -- Full BFS Coloring

A graph is bipartite if and only if it contains no odd-length cycles. The BFS coloring approach assigns alternating colors (0 and 1) to vertices. If a conflict is found (two adjacent vertices with the same color), the graph is not bipartite.

Bipartite Example: NOT Bipartite (odd cycle): Set A: {0, 2, 4} 0 -- 1 Set B: {1, 3, 5} | | 2 ---+ 0(A) -- 1(B) | | 0=Red, 1=Blue, 2=Red? 2(A) -- 3(B) But 2 is adjacent to 0 (also Red)! | | CONFLICT → Not bipartite 4(A) -- 5(B)

Disconnected Graphs

Always handle disconnected graphs! A graph with multiple components is bipartite only if every component is bipartite. The outer loop over all uncolored vertices ensures all components are checked.

Real-World Applications

Bipartite checking is used in: job assignment problems (workers vs tasks), graph coloring with 2 colors, matching algorithms (Hungarian method), and detecting conflicts in scheduling systems.

📋 Topological Sort (DFS)

A topological ordering of a DAG is a linear ordering of vertices such that for every edge u→v, u appears before v. Only possible for Directed Acyclic Graphs.

DFS Approach

Run DFS. Push each node to a stack AFTER all descendants are processed. Pop stack for the topological order.

⚡ Kahn's Algorithm (BFS Topological Sort)

Process nodes with in-degree 0 first (no dependencies). Remove their edges, enqueue newly zero-indegree nodes. If not all nodes processed, a cycle exists!

DAG: 5→2, 5→0, 4→0, 4→1, 2→3, 3→1 In-degrees: {0:2, 1:2, 2:1, 3:1, 4:0, 5:0} Queue: [4, 5] → process 4 → [5] → process 5 → [0, 2] → ... Result: [4, 5, 0, 2, 3, 1] (one valid topological order)

List<Integer> topoSortKahn() { int[] indegree = new int[V]; for (int u = 0; u < V; u++) for (int v : adj.get(u)) indegree[v]++; Queue<Integer> queue = new LinkedList<>(); for (int i = 0; i < V; i++) if (indegree[i] == 0) queue.offer(i); List<Integer> result = new ArrayList<>(); while (!queue.isEmpty()) { int u = queue.poll(); result.add(u); for (int v : adj.get(u)) { indegree[v]--; if (indegree[v] == 0) queue.offer(v); } } if (result.size() != V) return null; // cycle detected return result; }

fn topological_sort(&self) -> Option<Vec<usize>> { let mut in_degree = vec![0usize; self.num_vertices]; for u in 0..self.num_vertices { for &v in &self.adj[u] { in_degree[v] += 1; } } let mut queue: VecDeque<usize> = VecDeque::new(); for (v, &deg) in in_degree.iter().enumerate() { if deg == 0 { queue.push_back(v); } } let mut order = Vec::with_capacity(self.num_vertices); while let Some(u) = queue.pop_front() { order.push(u); for &v in &self.adj[u] { in_degree[v] -= 1; if in_degree[v] == 0 { queue.push_back(v); } } } if order.len() == self.num_vertices { Some(order) } else { None } }
