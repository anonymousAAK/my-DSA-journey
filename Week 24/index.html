⏱ Amortized Analysis

Amortized analysis computes the average cost per operation over a sequence of operations, even when individual operations may be expensive. Unlike average-case analysis, it guarantees worst-case bounds for the entire sequence.

Three Methods

MethodApproachKey Idea
AggregateTotal cost / n operationsSum all costs, divide by count
AccountingPrepay for future expensive opsCharge extra on cheap ops, "save" credit
PotentialΦ(state) = stored energyAmortized = actual + ΔΦ

📈 Dynamic Array (Amortized O(1) Push)

Classic example: ArrayList doubling. Each push is O(1) unless the array is full, then it costs O(n) to resize. Amortized cost per push: O(1) because resize happens rarely.

Push sequence (capacity doubles at powers of 2): push 1: cost 1 capacity 1 → 2 push 2: cost 1 + 2 (resize) capacity 2 → 4 push 3: cost 1 push 4: cost 1 + 4 (resize) capacity 4 → 8 push 5-8: cost 1 each push 9: cost 1 + 8 (resize) capacity 8 → 16 Total for n pushes: n + 1 + 2 + 4 + ... + n = n + (2n - 1) < 3n Amortized per push: 3n/n = O(1)
ResearchLevelTopics.java
class DynamicArray { int[] data; int size, capacity; DynamicArray() { data = new int[1]; size = 0; capacity = 1; } void push(int val) { if (size == capacity) { capacity *= 2; int[] newData = new int[capacity]; System.arraycopy(data, 0, newData, 0, size); data = newData; // O(n) resize, but rare } data[size++] = val; // O(1) } int get(int index) { return data[index]; } }
research_level.cpp
class DynamicArray { int* data; int sz, cap; public: DynamicArray() : sz(0), cap(1) { data = new int[1]; } ~DynamicArray() { delete[] data; } void push(int val) { if (sz == cap) { cap *= 2; int* newData = new int[cap]; std::copy(data, data + sz, newData); delete[] data; data = newData; } data[sz++] = val; } int get(int i) const { return data[i]; } };
research_level.py

🔗 Skip List

A probabilistic data structure: a sorted linked list with "express lanes." Each element is promoted to higher levels with probability p (typically 1/2). Expected O(log n) for search, insert, and delete — simpler to implement than balanced BSTs.

Level 3: HEAD ──────────────────────────── 50 ── NIL Level 2: HEAD ──────── 20 ──────────────── 50 ── NIL Level 1: HEAD ── 10 ── 20 ──── 35 ──────── 50 ── NIL Level 0: HEAD ── 10 ── 20 ── 25 ── 35 ── 42 ── 50 ── NIL Search 35: Start at top level, skip forward, drop down
Where It's Used
Redis sorted sets, LevelDB, Java's ConcurrentSkipListMap. Skip lists offer simpler concurrent implementations than balanced BSTs since they don't require complex tree rotations.
ResearchLevelTopics.java
class SkipList { static final int MAX_LEVEL = 16; static final double P = 0.5; int level; Node head; static class Node { int val; Node[] next; Node(int v, int lvl) { val = v; next = new Node[lvl + 1]; } } SkipList() { head = new Node(Integer.MIN_VALUE, MAX_LEVEL); level = 0; } int randomLevel() { int lvl = 0; while (Math.random() < P && lvl < MAX_LEVEL) lvl++; return lvl; } boolean search(int target) { Node curr = head; for (int i = level; i >= 0; i--) while (curr.next[i] != null && curr.next[i].val < target) curr = curr.next[i]; curr = curr.next[0]; return curr != null && curr.val == target; } void insert(int val) { Node[] update = new Node[MAX_LEVEL + 1]; Node curr = head; for (int i = level; i >= 0; i--) { while (curr.next[i] != null && curr.next[i].val < val) curr = curr.next[i]; update[i] = curr; } int newLvl = randomLevel(); if (newLvl > level) { for (int i = level + 1; i <= newLvl; i++) update[i] = head; level = newLvl; } Node node = new Node(val, newLvl); for (int i = 0; i <= newLvl; i++) { node.next[i] = update[i].next[i]; update[i].next[i] = node; } } }

🌸 Bloom Filter

A space-efficient probabilistic set. Supports add(x) and mightContain(x). False positives are possible (says "yes" when element is not present), but no false negatives (if it says "no", element is definitely absent).

add("cat"): hash1("cat") = 2, hash2("cat") = 5, hash3("cat") = 9 Bit array: [0 0 1 0 0 1 0 0 0 1 0 0] check("dog"): hash1("dog") = 2, hash2("dog") = 7, hash3("dog") = 9 Positions: 2=1, 7=0, 9=1 Result: NOT in set (bit 7 is 0) check("fox"): hash1("fox") = 2, hash2("fox") = 5, hash3("fox") = 9 All bits set! Result: MAYBE in set (false positive possible)
ResearchLevelTopics.java
class BloomFilter { boolean[] bits; int size, numHashes; BloomFilter(int size, int numHashes) { this.bits = new boolean[size]; this.size = size; this.numHashes = numHashes; } void add(String item) { for (int i = 0; i < numHashes; i++) bits[hash(item, i)] = true; } boolean mightContain(String item) { for (int i = 0; i < numHashes; i++) if (!bits[hash(item, i)]) return false; return true; // might be false positive } private int hash(String item, int seed) { int h = item.hashCode() + seed * 31; return Math.abs(h) % size; } }
research_level.cpp
class BloomFilter { std::vector<bool> bits; int m, k; size_t hash_fn(const std::string& s, int seed) { size_t h = std::hash<std::string>{}(s) + seed * 0x9e3779b9; return h % m; } public: BloomFilter(int size, int numHashes) : m(size), k(numHashes), bits(size, false) {} void add(const std::string& item) { for (int i = 0; i < k; ++i) bits[hash_fn(item, i)] = true; } bool might_contain(const std::string& item) { for (int i = 0; i < k; ++i) if (!bits[hash_fn(item, i)]) return false; return true; } };

use std::collections::hash_map::DefaultHasher; use std::hash::{Hash, Hasher}; struct BloomFilter { bits: Vec<bool>, size: usize, num_hashes: usize, } impl BloomFilter { fn new(size: usize, num_hashes: usize) -> Self { BloomFilter { bits: vec![false; size], size, num_hashes } } fn hash_item(&self, item: &str, seed: usize) -> usize { let mut hasher = DefaultHasher::new(); item.hash(&mut hasher); seed.hash(&mut hasher); hasher.finish() as usize % self.size } fn add(&mut self, item: &str) { for i in 0..self.num_hashes { let idx = self.hash_item(item, i); self.bits[idx] = true; } } fn might_contain(&self, item: &str) -> bool { (0..self.num_hashes).all(|i| self.bits[self.hash_item(item, i)]) } }

🌊 Reservoir Sampling

Uniformly sample k items from a stream of unknown size N. Each item ends up in the sample with equal probability k/N. Time: O(N), Space: O(k) — single pass!

Algorithm (k=1)
Keep the first element. For the i-th element (1-indexed), replace the kept element with probability 1/i. After seeing all N elements, each element has exactly 1/N probability of being chosen.
ResearchLevelTopics.java
static int[] reservoirSample(int[] stream, int k) { int[] reservoir = new int[k]; Random rand = new Random(); for (int i = 0; i < stream.length; i++) { if (i < k) { reservoir[i] = stream[i]; } else { int j = rand.nextInt(i + 1); if (j < k) reservoir[j] = stream[i]; } } return reservoir; }
research_level.cpp
std::vector<int> reservoir_sample(const std::vector<int>& stream, int k) { std::vector<int> reservoir(stream.begin(), stream.begin() + k); std::mt19937 rng(42); for (int i = k; i < (int)stream.size(); ++i) { int j = std::uniform_int_distribution<int>(0, i)(rng); if (j < k) reservoir[j] = stream[i]; } return reservoir; }
research_level.py

🧠 NP-Completeness

The most fundamental open question in computer science: does P = NP?

ClassDefinitionExamples
PSolvable in polynomial time O(nk)Sorting, shortest path, MST
NPSolution verifiable in polynomial timeAll of P, plus SAT, TSP, etc.
NP-CompleteIn NP & every NP problem reduces to it3-SAT, Vertex Cover, Subset Sum
NP-HardAt least as hard as NP-CompleteHalting problem, TSP optimization

Famous NP-Complete Problems

  • 3-SAT: Boolean satisfiability with clauses of exactly 3 literals
  • Traveling Salesman (decision): Is there a tour of length ≤ k?
  • Vertex Cover: Can we cover all edges with ≤ k vertices?
  • Hamiltonian Cycle: Does a cycle visiting every vertex exactly once exist?
  • Subset Sum: Does a subset summing to exactly S exist?
  • Graph Coloring: Can we color with ≤ k colors so no adjacent vertices match?
Key Implication
If you can solve ANY NP-Complete problem in polynomial time, you can solve ALL of them (P = NP). Most experts believe P ≠ NP, meaning no efficient exact algorithms exist for these problems.

🎯 Approximation Algorithms

When exact solutions are intractable (NP-Complete), find solutions guaranteed to be within a factor of optimal. The approximation ratio ρ bounds how far the solution can be from OPT.

ProblemApproximation RatioAlgorithm
Vertex Cover2Take both endpoints of maximal matching
Set CoverO(log n)Greedy: pick set covering most uncovered
TSP (metric)3/2Christofides algorithm (MST-based)

💥 2-Approximation Vertex Cover

Find a vertex cover (set of vertices touching all edges) at most 2× the optimal size. Pick any uncovered edge, add both endpoints, repeat.

NPCompletenessAndApproximation.java
static Set<Integer> vertexCover2Approx(int V, int[][] edges) { Set<Integer> cover = new HashSet<>(); boolean[] edgeCovered = new boolean[edges.length]; for (int i = 0; i < edges.length; i++) { if (!edgeCovered[i]) { int u = edges[i][0], v = edges[i][1]; if (!cover.contains(u) && !cover.contains(v)) { cover.add(u); cover.add(v); // Mark all edges incident to u or v for (int j = i; j < edges.length; j++) if (edges[j][0] == u || edges[j][0] == v || edges[j][1] == u || edges[j][1] == v) edgeCovered[j] = true; } } } return cover; // size <= 2 * OPT }
research_level.cpp
std::set<int> vertex_cover_2approx( int V, const std::vector<std::pair<int,int>>& edges) { std::set<int> cover; std::vector<bool> covered(edges.size(), false); for (size_t i = 0; i < edges.size(); ++i) { auto [u, v] = edges[i]; if (cover.count(u) == 0 && cover.count(v) == 0) { cover.insert(u); cover.insert(v); for (size_t j = i; j < edges.size(); ++j) { auto [a, b] = edges[j]; if (a == u || a == v || b == u || b == v) covered[j] = true; } } } return cover; }

📦 Greedy Set Cover

Given a universe U and collection of sets, find minimum sets whose union covers U. NP-Complete for exact solution, but greedy achieves O(log n) approximation.

Greedy Strategy
Always pick the set that covers the most currently-uncovered elements. Repeat until everything is covered. Approximation ratio: H(n) ≈ ln(n) where n = |U|.
NPCompletenessAndApproximation.java
static List<Set<Integer>> greedySetCover( Set<Integer> universe, List<Set<Integer>> sets) { Set<Integer> uncovered = new HashSet<>(universe); List<Set<Integer>> chosen = new ArrayList<>(); while (!uncovered.isEmpty()) { // Pick set that covers the most uncovered elements Set<Integer> best = null; int bestCount = 0; for (Set<Integer> s : sets) { int count = 0; for (int e : s) if (uncovered.contains(e)) count++; if (count > bestCount) { bestCount = count; best = s; } } if (best == null) break; chosen.add(best); uncovered.removeAll(best); } return chosen; }
research_level.cpp
std::vector<int> greedy_set_cover( const std::set<int>& universe, const std::vector<std::set<int>>& sets) { std::set<int> uncovered = universe; std::vector<int> chosen; while (!uncovered.empty()) { int best = -1, bestCount = 0; for (int i = 0; i < (int)sets.size(); ++i) { int count = 0; for (int e : sets[i]) if (uncovered.count(e)) ++count; if (count > bestCount) { bestCount = count; best = i; } } if (best == -1) break; chosen.push_back(best); for (int e : sets[best]) uncovered.erase(e); } return chosen; }
research_level.py
def greedy_set_cover(universe, sets): uncovered = set(universe) chosen = [] while uncovered: best = max(sets, key=lambda s: len(s & uncovered)) chosen.append(best) uncovered -= best return chosen
research_level.rs
fn greedy_set_cover( universe: &HashSet<usize>, sets: &[HashSet<usize>], ) -> Vec<usize> { let mut uncovered = universe.clone(); let mut chosen = Vec::new(); while !uncovered.is_empty() { let best = (0..sets.len()) .max_by_key(|&i| sets[i].intersection(&uncovered).count()) .unwrap(); chosen.push(best); for &e in &sets[best] { uncovered.remove(&e); } } chosen }
