Rate Limiting - Token Bucket / Leaky Bucket / Sliding Window

WEEK 29 - WEB (JS) ADVANCED TOPICS

CONCEPT: Cap how often an action may occur: token-bucket (bursts up to capacity, average rate), leaky-bucket (steady outflow), sliding-window log (exact count over a time window).

KEY POINTS: Refill formula = elapsed * rate. Use performance.now() for monotonic timing.

ALGORITHM: token: refill + consume. leaky: drain + admit if room. sliding: drop old timestamps + check count.

JS-SPECIFIC NOTES: performance.now() returns ms; we convert to seconds for the rate math.

DRY RUN: TokenBucket(5, 2 tok/s) -> 5 burst allowed, then rejected; 500 ms later 1 more allowed.

COMPLEXITY: Token/Leaky O(1); Sliding O(window).
