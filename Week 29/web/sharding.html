Sharding Strategies

WEEK 29 - WEB (JS) ADVANCED TOPICS

CONCEPT: Partition data across N nodes via range, hash, or directory schemes; each trades range-query support, balance, and rebalance cost.

KEY POINTS: Range supports scans; Hash spreads uniformly; Directory offers per-key control.

ALGORITHM: Range: binary search by key. Hash: hash(key) mod N. Directory: explicit map lookup.

JS-SPECIFIC NOTES: Use SubtleCrypto (async) for cross-process stable hashes; we use a tiny FNV-1a 32-bit hash for synchronous demo simplicity.

DRY RUN: Range boundaries ["c","m","t"]; "apple"->0, "cat"->1, "mango"->2, "tiger"->3.

COMPLEXITY: Range O(log shards); Hash O(L); Dir O(1).
