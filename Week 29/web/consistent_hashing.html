Consistent Hashing with Virtual Nodes

WEEK 29 - WEB (JS) ADVANCED TOPICS

CONCEPT: Map nodes and keys onto a 32-bit hash ring; each key belongs to the next node clockwise. Virtual nodes per server balance load and limit remapping when nodes leave/join.

KEY POINTS: ~1/N keys move on add/remove. O(log NV) lookup via binary search on a sorted hash array.

ALGORITHM: add: insert V hashes for each VN. remove: erase those. get: upper_bound of hash(key); wrap to 0.

JS-SPECIFIC NOTES: Use a small FNV-1a synchronously (SubtleCrypto is async); store hash -> node in a Map plus a sorted array of hashes for binary search.

DRY RUN: Add 3 servers @ 150 VNs each. Look up keys before and after removing server-B; usually only ~1/3 of keys move.

COMPLEXITY: add/remove O(V log NV); get O(log NV).
