Caching - LRU / LFU / TTL

WEEK 29 - WEB (JS) ADVANCED TOPICS

CONCEPT: Bounded cache with an eviction policy. LRU drops least recently used; LFU drops least frequently used; TTL drops after a deadline.

KEY POINTS: O(1) get/put with Map's insertion-order property (LRU); per-frequency Maps (LFU); lazy expiry on access (TTL).

ALGORITHM: LRU: delete + re-insert on touch (insertion-ordered Map). LFU: bump freq + bucket migration. TTL: compare performance.now() to deadline.

JS-SPECIFIC NOTES: Map preserves insertion order, so LRU is trivially implementable without a doubly-linked list.

DRY RUN: Demo plays the canonical LeetCode 146/460 sequences and a TTL expiry round-trip.

COMPLEXITY: All ops O(1) amortised.
