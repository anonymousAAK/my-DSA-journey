Message Queues - FIFO / Priority / Delayed / Pub-Sub

WEEK 29 - WEB (JS) ADVANCED TOPICS

CONCEPT: Decouple producers and consumers. FIFO for order; Priority for urgency; Delayed for scheduled delivery; Pub/Sub for broadcast.

KEY POINTS: Heap for priority/delayed; Array for FIFO; Map of topic -> callbacks for pub/sub.

ALGORITHM: standard data-structure operations; for delayed queue poll only when deadline reached.

JS-SPECIFIC NOTES: JS has no built-in heap; we hand-roll a small one. performance.now() drives the delayed deadlines.

DRY RUN: Priority {low,med,high} -> pop high, med, low. Delayed put('A', 50 ms); poll now -> null; setTimeout 60 ms -> 'A'.

COMPLEXITY: FIFO O(1), priority/delayed O(log n), pub/sub O(subs/publish).
