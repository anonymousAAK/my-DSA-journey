📚 Stack Implementation

A stack is a LIFO (Last In, First Out) data structure. Think of a stack of plates -- you always add and remove from the top.

Core Operations (all O(1))

OperationDescriptionTime
push(x)Add element to the topO(1)
pop()Remove and return the top elementO(1)
peek() / top()View the top element without removingO(1)
isEmpty()Check if the stack is emptyO(1)

Push & Pop Visualization

push(10), push(20), push(30): TOP | | 30 | ← push(30) | 20 | | 10 | +----+ pop() returns 30: TOP | | 20 | | 10 | +----+ pop() returns 20: TOP | | 10 | +----+
Java Note
In Java, prefer Deque<Integer> (implemented by ArrayDeque) over the legacy Stack class. ArrayDeque is faster, more cache-friendly, and has a cleaner API.
StackImplementation.java
static class ArrayStack { private int[] data; private int top; ArrayStack(int capacity) { data = new int[capacity]; top = -1; } void push(int x) { if (top == data.length - 1) throw new RuntimeException("Stack overflow"); data[++top] = x; } int pop() { if (isEmpty()) throw new RuntimeException("Stack underflow"); return data[top--]; } int peek() { if (isEmpty()) throw new RuntimeException("Stack is empty"); return data[top]; } boolean isEmpty() { return top == -1; } int size() { return top + 1; } }
stacks.cpp
template <typename T> class Stack { vector<T> data_; public: void push(const T& val) { data_.push_back(val); } void pop() { if (data_.empty()) throw runtime_error("Stack underflow"); data_.pop_back(); } T& top() { if (data_.empty()) throw runtime_error("Stack is empty"); return data_.back(); } bool empty() const { return data_.empty(); } size_t size() const { return data_.size(); } };
stacks.py
class Stack: """LIFO stack backed by a Python list.""" def __init__(self) -> None: self._data: list[int] = [] def push(self, val: int) -> None: self._data.append(val) def pop(self) -> int: if self.is_empty(): raise IndexError("pop from empty stack") return self._data.pop() def peek(self) -> int: if self.is_empty(): raise IndexError("peek from empty stack") return self._data[-1] def is_empty(self) -> bool: return len(self._data) == 0 def size(self) -> int: return len(self._data)
stacks.rs
struct Stack { data: Vec<i32>, } impl Stack { fn new() -> Self { Stack { data: Vec::new() } } fn push(&mut self, value: i32) { self.data.push(value); } fn pop(&mut self) -> Option<i32> { self.data.pop() } fn peek(&self) -> Option<&i32> { self.data.last() } fn is_empty(&self) -> bool { self.data.is_empty() } fn len(&self) -> usize { self.data.len() } }
Rust Note
Vec<T> is the natural stack in Rust: push() and pop() are O(1) amortized. pop() returns Option<T>, forcing you to handle the empty-stack case at compile time -- no null pointer exceptions.

✔ Balanced Parentheses

Check if a string of brackets (){}[] is properly balanced. This is a classic stack application: push opening brackets, pop and match when encountering closing brackets.

Algorithm

  1. For each character in the string:
  2. If it is an opening bracket ( [ {, push it onto the stack
  3. If it is a closing bracket ) ] }, pop from the stack and check if it matches
  4. If the stack is empty when we need to pop, or the match fails, return false
  5. At the end, the stack must be empty (all brackets matched)
Input: "({[]})" Step 1: '(' → push Stack: [(] Step 2: '{' → push Stack: [(, {] Step 3: '[' → push Stack: [(, {, [] Step 4: ']' → pop '[' ✓ Stack: [(, {] Step 5: '}' → pop '{' ✓ Stack: [(] Step 6: ')' → pop '(' ✓ Stack: [] Stack empty → BALANCED Input: "(]" Step 1: '(' → push Stack: [(] Step 2: ']' → pop '(' ≠ matching '[' → NOT BALANCED
StackImplementation.java
static boolean isBalanced(String s) { Deque<Character> stack = new ArrayDeque<>(); for (char c : s.toCharArray()) { if (c == '(' || c == '{' || c == '[') { stack.push(c); } else { if (stack.isEmpty()) return false; char top = stack.pop(); if (c == ')' && top != '(') return false; if (c == '}' && top != '{') return false; if (c == ']' && top != '[') return false; } } return stack.isEmpty(); }
stacks.cpp
bool isBalanced(const string& s) { stack<char> st; unordered_map<char, char> match = { {')', '('}, {'}', '{'}, {']', '['} }; for (char c : s) { if (c == '(' || c == '{' || c == '[') { st.push(c); } else if (match.count(c)) { if (st.empty() || st.top() != match[c]) return false; st.pop(); } } return st.empty(); }
stacks.py
def is_balanced(s: str) -> bool: matching = {")": "(", "]": "[", "}": "{"} stack = [] for ch in s: if ch in "([{": stack.append(ch) elif ch in ")]}": if not stack or stack[-1] != matching[ch]: return False stack.pop() return len(stack) == 0
stacks.rs
fn is_balanced(s: &str) -> bool { let mut stack: Vec<char> = Vec::new(); let matches: HashMap<char, char> = [(')', '('), (']', '['), ('}', '{')] .iter().cloned().collect(); for ch in s.chars() { match ch { '(' | '[' | '{' => stack.push(ch), ')' | ']' | '}' => { match (stack.pop(), matches.get(&ch)) { (Some(top), Some(&expected)) if top == expected => {} _ => return false, } } _ => {} } } stack.is_empty() }

📈 Next Greater Element (Monotonic Stack)

For each element in an array, find the next element to the right that is greater than it. If none exists, the result is -1.

The Monotonic Stack Technique

Maintain a monotonically decreasing stack of indices. When we encounter an element larger than the stack top, that element is the "next greater" for all smaller elements on top of the stack.

Array: [4, 5, 2, 10, 8] i=0: arr[0]=4, stack empty, push 0 Stack: [0] i=1: arr[1]=5 > arr[0]=4 → NGE[0]=5, pop Stack: [] push 1 Stack: [1] i=2: arr[2]=2, 2 < 5, push 2 Stack: [1, 2] i=3: arr[3]=10 > arr[2]=2 → NGE[2]=10 arr[3]=10 > arr[1]=5 → NGE[1]=10 push 3 Stack: [3] i=4: arr[4]=8, 8 < 10, push 4 Stack: [3, 4] Remaining: NGE[3]=-1, NGE[4]=-1 Result: [5, 10, 10, -1, -1]
Key Insight
Each element is pushed and popped from the stack at most once, so the total work is O(n) despite the nested while loop. This is the hallmark of the monotonic stack pattern.
StackImplementation.java
static int[] nextGreaterElement(int[] arr) { int n = arr.length; int[] result = new int[n]; java.util.Arrays.fill(result, -1); Deque<Integer> stack = new ArrayDeque<>(); // stores INDICES for (int i = 0; i < n; i++) { while (!stack.isEmpty() && arr[i] > arr[stack.peek()]) { int idx = stack.pop(); result[idx] = arr[i]; } stack.push(i); } return result; }
stacks.cpp
vector<int> nextGreaterElement(const vector<int>& arr) { int n = arr.size(); vector<int> result(n, -1); stack<int> st; // stores indices for (int i = 0; i < n; ++i) { while (!st.empty() && arr[st.top()] < arr[i]) { result[st.top()] = arr[i]; st.pop(); } st.push(i); } return result; } // Circular variant: traverse the array twice vector<int> nextGreaterElementCircular(const vector<int>& arr) { int n = arr.size(); vector<int> result(n, -1); stack<int> st; for (int i = 0; i < 2 * n; ++i) { int idx = i % n; while (!st.empty() && arr[st.top()] < arr[idx]) { result[st.top()] = arr[idx]; st.pop(); } if (i < n) st.push(i); } return result; }
stacks.py

stacks.rs
fn next_greater_element(arr: &[i32]) -> Vec<Option<i32>> { let n = arr.len(); let mut result: Vec<Option<i32>> = vec![None; n]; let mut stack: Vec<usize> = Vec::new(); for i in 0..n { while let Some(&top_idx) = stack.last() { if arr[top_idx] < arr[i] { stack.pop(); result[top_idx] = Some(arr[i]); } else { break; } } stack.push(i); } result }

📉 Min Stack -- O(1) getMin()

A stack that supports push, pop, peek, and getMin() -- all in O(1) time. The trick: maintain a second stack that tracks the current minimum at each level.

How It Works

Operations: push(5), push(3), push(7), push(2), pop(), pop() Main Stack: Min Stack: push(5): [5] [5] min = 5 push(3): [5,3] [5,3] min = 3 push(7): [5,3,7] [5,3,3] min = 3 (7 > 3, min stays 3) push(2): [5,3,7,2] [5,3,3,2] min = 2 pop(2): [5,3,7] [5,3,3] min = 3 (back to 3!) pop(7): [5,3] [5,3] min = 3 (still 3)
Key Insight
The min stack stores the minimum at each "level" of the main stack. When we push, the new minimum is min(new_value, current_min). When we pop, we pop from both stacks, automatically restoring the previous minimum.
StackImplementation.java
static class MinStack { Deque<Integer> stack = new ArrayDeque<>(); Deque<Integer> minStack = new ArrayDeque<>(); void push(int x) { stack.push(x); int min = minStack.isEmpty() ? x : Math.min(x, minStack.peek()); minStack.push(min); } void pop() { stack.pop(); minStack.pop(); } int peek() { return stack.peek(); } int getMin() { return minStack.peek(); } }
stacks.cpp
class MinStack { stack<int> mainStack_; stack<int> minStack_; public: void push(int val) { mainStack_.push(val); if (minStack_.empty() || val <= minStack_.top()) { minStack_.push(val); } } void pop() { if (mainStack_.top() == minStack_.top()) { minStack_.pop(); } mainStack_.pop(); } int top() const { return mainStack_.top(); } int getMin() const { return minStack_.top(); } };
stacks.py
class MinStack: def __init__(self) -> None: self._stack = [] self._min_stack = [] def push(self, val: int) -> None: self._stack.append(val) current_min = min(val, self._min_stack[-1]) if self._min_stack else val self._min_stack.append(current_min) def pop(self) -> int: self._min_stack.pop() return self._stack.pop() def top(self) -> int: return self._stack[-1] def get_min(self) -> int: return self._min_stack[-1]
stacks.rs
struct MinStack { data: Vec<i32>, mins: Vec<i32>, } impl MinStack { fn new() -> Self { MinStack { data: Vec::new(), mins: Vec::new() } } fn push(&mut self, value: i32) { self.data.push(value); let new_min = match self.mins.last() { Some(&current_min) => value.min(current_min), None => value, }; self.mins.push(new_min); } fn pop(&mut self) -> Option<i32> { self.mins.pop(); self.data.pop() } fn top(&self) -> Option<&i32> { self.data.last() } fn get_min(&self) -> Option<&i32> { self.mins.last() } }

🔢 Postfix (RPN) Expression Evaluation

Reverse Polish Notation (postfix) places operators after their operands: 3 4 + means 3 + 4 = 7. Stacks make evaluation trivial -- push operands, pop two when you see an operator, compute, and push the result.

Step-by-Step Example

Expression: "3 4 + 2 *" (meaning: (3 + 4) * 2 = 14) Token "3": push 3 Stack: [3] Token "4": push 4 Stack: [3, 4] Token "+": pop 4 & 3 Stack: [] push 3+4=7 Stack: [7] Token "2": push 2 Stack: [7, 2] Token "*": pop 2 & 7 Stack: [] push 7*2=14 Stack: [14] Result: 14
Order Matters for - and /
When popping for subtraction and division, the first popped value is the right operand (b) and the second is the left operand (a). Compute a - b and a / b, not the reverse.
StackImplementation.java
static int evalPostfix(String expr) { Deque<Integer> stack = new ArrayDeque<>(); for (String token : expr.split("\\s+")) { switch (token) { case "+": stack.push(stack.pop() + stack.pop()); break; case "*": stack.push(stack.pop() * stack.pop()); break; case "-": { int b = stack.pop(), a = stack.pop(); stack.push(a - b); break; } case "/": { int b = stack.pop(), a = stack.pop(); stack.push(a / b); break; } default: stack.push(Integer.parseInt(token)); } } return stack.pop(); }
stacks.cpp
int evalPostfix(const vector<string>& tokens) { stack<int> st; for (const auto& token : tokens) { if (token == "+" || token == "-" || token == "*" || token == "/") { int b = st.top(); st.pop(); int a = st.top(); st.pop(); int result; if (token == "+") result = a + b; else if (token == "-") result = a - b; else if (token == "*") result = a * b; else result = a / b; st.push(result); } else { st.push(stoi(token)); } } return st.top(); }
stacks.py
def eval_postfix(tokens): stack = [] for token in tokens: if token in ("+", "-", "*", "/"): b = stack.pop() a = stack.pop() if token == "+": stack.append(a + b) elif token == "-": stack.append(a - b) elif token == "*": stack.append(a * b) else: stack.append(int(a / b)) # truncate toward zero else: stack.append(int(token)) return stack[0]
stacks.rs
fn eval_postfix(expression: &str) -> Option<i32> { let mut stack: Vec<i32> = Vec::new(); for token in expression.split_whitespace() { match token { "+" | "-" | "*" | "/" => { let b = stack.pop()?; let a = stack.pop()?; let result = match token { "+" => a + b, "-" => a - b, "*" => a * b, "/" => { if b == 0 { return None; } a / b } _ => unreachable!(), }; stack.push(result); } _ => { stack.push(token.parse().ok()?); } } } if stack.len() == 1 { stack.pop() } else { None } }

When to Use a Stack
Look for these signals: (1) Last-in-first-out processing order. (2) Matching pairs (parentheses, tags). (3) Nearest previous/next element problems. (4) Undo functionality. (5) Converting recursion to iteration. (6) Expression parsing.
