Methods & Recursion

Learn how to decompose problems into reusable methods and master recursion -- the elegant technique where functions call themselves to solve progressively smaller subproblems.

Beginner-Intermediate ~6 hours estimated 6 topics covered

🎯 Learning Objectives

  • Understand method syntax: parameters, return types, and scope
  • Learn method overloading and compile-time polymorphism
  • Master the two pillars of recursion: base case and recursive case
  • Implement classic recursion problems: factorial, Fibonacci, Tower of Hanoi
  • Compare naive recursion vs memoization vs iteration for Fibonacci
  • Recognize common recursion patterns: linear, binary, tail, and mutual recursion
  • Understand the call stack and how recursion uses memory

🔧 Method Basics

A method (function) is a named, reusable block of code. Methods promote reusability, readability, and maintainability. In Java, every method has a return type, a name, and an optional parameter list.

Key Concepts

  • void means the method returns nothing
  • return exits a method and optionally passes a value back
  • Each method call creates a new stack frame on the call stack
  • Java passes primitives by value -- the original variable is never modified
Call Stack

Each method call pushes a frame onto the stack. When the method returns, its frame is popped. This LIFO (Last-In, First-Out) behavior is what makes recursion possible.

MethodBasics.java
// Method with no parameters and no return value static void greet() { System.out.println("Hello from a method!"); } // Method with parameters AND a return value static int add(int a, int b) { return a + b; } // Pass-by-value demo: original is unaffected static void tryToChange(int n) { n = 999; // Only changes the LOCAL copy System.out.println("Inside tryToChange: n = " + n); } public static void main(String[] args) { greet(); // Output: Hello from a method! int result = add(5, 6); System.out.println("add(5, 6) = " + result); // 11 int x = 42; tryToChange(x); System.out.println("After tryToChange: x = " + x); // Still 42 }

// Recursive factorial // Time: O(n) Space: O(n) call stack long long factorialRecursive(int n) { if (n <= 1) return 1; return static_cast<long long>(n) * factorialRecursive(n - 1); } // Iterative factorial // Time: O(n) Space: O(1) long long factorialIterative(int n) { long long result = 1; for (int i = 2; i <= n; ++i) { result *= i; } return result; }

def factorial_iterative(n: int) -> int: """Return n! using a simple loop. Time: O(n), Space: O(1)""" result = 1 for i in range(2, n + 1): result *= i return result def factorial_recursive(n: int) -> int: """Return n! using head recursion. Time: O(n), Space: O(n)""" if n <= 1: return 1 return n * factorial_recursive(n - 1) def factorial_tail(n: int, accumulator: int = 1) -> int: """Return n! using tail recursion via an accumulator.""" if n <= 1: return accumulator return factorial_tail(n - 1, n * accumulator)
recursion.rs (functions)
/// Computes `n!` iteratively. Time: O(n), Space: O(1) fn factorial_iterative(n: u64) -> u64 { (1..=n).fold(1u64, |acc, x| acc * x) } /// Computes `n!` recursively. Time: O(n), Space: O(n) fn factorial_recursive(n: u64) -> u64 { match n { 0 | 1 => 1, _ => n * factorial_recursive(n - 1), } }

🔄 Method Overloading

Method overloading means multiple methods with the same name but different parameter lists (different number, type, or order of parameters). The compiler picks the right version at compile time -- this is called compile-time polymorphism or static dispatch.

Rules

  • Different parameter count OR different parameter types
  • Return type alone does NOT distinguish overloaded methods
MethodOverloading.java
// Version 1: two ints static int add(int a, int b) { return a + b; } // Version 2: three ints static int add(int a, int b, int c) { return a + b + c; } // Version 3: two doubles static double add(double a, double b) { return a + b; } // Overloaded print utility for different array types static void printArray(int[] arr) { System.out.print("int[]: "); for (int x : arr) System.out.print(x + " "); System.out.println(); } static void printArray(double[] arr) { System.out.print("double[]: "); for (double x : arr) System.out.print(x + " "); System.out.println(); } public static void main(String[] args) { System.out.println(add(2, 3)); // calls version 1 System.out.println(add(1, 2, 3)); // calls version 2 System.out.println(add(1.5, 2.5)); // calls version 3 }
DSA Relevance

Many utility methods (print, compare, swap) benefit from overloading to handle different data types without renaming. Java's Arrays.sort() is itself overloaded for int[], double[], Object[], etc.

🔁 Recursion Basics

A recursive function calls itself to solve a smaller version of the same problem. Every recursive function needs two things:

  1. Base Case -- the condition that stops recursion (prevents infinite loop)
  2. Recursive Case -- the function calling itself with a smaller/simpler input
Common Mistake

Forgetting the base case leads to infinite recursion and a StackOverflowError. Always define your stopping condition first!

Step-by-Step: Factorial

factorial(5) = 5 * factorial(4) = 5 * 4 * factorial(3) = 5 * 4 * 3 * factorial(2) = 5 * 4 * 3 * 2 * factorial(1) = 5 * 4 * 3 * 2 * 1 ← base case reached! = 120
RecursionBasics.java
// Print N down to 1 static void printDesc(int n) { if (n == 0) return; // base case System.out.print(n + " "); printDesc(n - 1); // recursive call with smaller n } // Factorial: N! = N * (N-1)! // Time: O(n), Space: O(n) call stack static long factorial(int n) { if (n <= 1) return 1; // base case return n * factorial(n - 1); } // Sum of first N natural numbers static int sum(int n) { if (n == 0) return 0; return n + sum(n - 1); } // Fast Power: O(log exp) // If exp is even: base^exp = (base^(exp/2))^2 // If exp is odd: base^exp = base * base^(exp-1) static long fastPower(long base, int exp) { if (exp == 0) return 1; if (exp % 2 == 0) { long half = fastPower(base, exp / 2); return half * half; } else { return base * fastPower(base, exp - 1); } }
recursion.cpp
// Recursive fast power // Time: O(log n) Space: O(log n) call stack long long fastPowRecursive(long long base, long long exp, long long mod) { if (exp == 0) return 1; long long half = fastPowRecursive(base, exp / 2, mod); half = (half * half) % mod; if (exp % 2 == 1) { half = (half * (base % mod)) % mod; } return half; } // Iterative fast power // Time: O(log n) Space: O(1) long long fastPowIterative(long long base, long long exp, long long mod) { long long result = 1; base %= mod; while (exp > 0) { if (exp & 1) { result = (result * base) % mod; } exp >>= 1; base = (base * base) % mod; } return result; }
recursion.py
def power_fast(base: float, exp: int) -> float: """Compute base^exp via fast (binary) exponentiation. Time: O(log n), Space: O(log n)""" if exp < 0: return 1.0 / power_fast(base, -exp) if exp == 0: return 1.0 if exp % 2 == 0: half = power_fast(base, exp // 2) return half * half return base * power_fast(base, exp - 1)
recursion.rs
/// Recursive fast power. Time: O(log exp), Space: O(log exp) fn fast_power_recursive(base: i64, exp: u32) -> i64 { match exp { 0 => 1, _ if exp % 2 == 0 => { let half = fast_power_recursive(base, exp / 2); half * half } _ => { let half = fast_power_recursive(base, exp / 2); base * half * half } } } /// Iterative fast power. Time: O(log exp), Space: O(1) fn fast_power(mut base: i64, mut exp: u32) -> i64 { let mut result: i64 = 1; while exp > 0 { if exp % 2 == 1 { result *= base; } base *= base; exp /= 2; } result }

🍀 Fibonacci: Recursion vs Memoization vs Iteration

The Fibonacci sequence (0, 1, 1, 2, 3, 5, 8, 13, 21, ...) is defined as fib(n) = fib(n-1) + fib(n-2). It is the classic introduction to Dynamic Programming thinking.

Three Approaches

ApproachTimeSpaceNotes
Naive RecursionO(2^n)O(n)Recomputes same values many times
Memoization (Top-Down DP)O(n)O(n)Cache results of subproblems
Iterative (Bottom-Up DP)O(n)O(1)Most efficient approach
Key Insight

fibNaive(5) makes 15 recursive calls. With memoization, it makes only 9 calls. For fib(45), naive recursion takes ~30 seconds while memoization is instant. Always think about overlapping subproblems!

FibonacciRecursion.java
// APPROACH 1: Naive recursion -- O(2^n) time static long fibNaive(int n) { if (n <= 1) return n; return fibNaive(n - 1) + fibNaive(n - 2); } // APPROACH 2: Memoization -- O(n) time, O(n) space static long[] memo = new long[100]; static long fibMemo(int n) { if (n <= 1) return n; if (memo[n] != -1) return memo[n]; // already computed memo[n] = fibMemo(n - 1) + fibMemo(n - 2); return memo[n]; } // APPROACH 3: Iterative -- O(n) time, O(1) space static long fibIterative(int n) { if (n <= 1) return n; long prev2 = 0, prev1 = 1; for (int i = 2; i <= n; i++) { long curr = prev1 + prev2; prev2 = prev1; prev1 = curr; } return prev1; }
recursion.cpp
// Memoized fibonacci (top-down DP) // Time: O(n) Space: O(n) memo table + O(n) call stack long long fibMemo(int n, unordered_map<int, long long>& memo) { if (n <= 1) return n; if (auto it = memo.find(n); it != memo.end()) { return it->second; // C++17 init-statement in if } memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo); return memo[n]; } // Iterative fibonacci (bottom-up) // Time: O(n) Space: O(1) long long fibIterative(int n) { if (n <= 1) return n; long long prev2 = 0, prev1 = 1; for (int i = 2; i <= n; ++i) { long long curr = prev1 + prev2; prev2 = prev1; prev1 = curr; } return prev1; }
recursion.py
from functools import lru_cache @lru_cache(maxsize=None) def fibonacci_memo(n: int) -> int: """Return fib(n) with automatic memoization via @lru_cache. Time: O(n), Space: O(n)""" if n <= 1: return n return fibonacci_memo(n - 1) + fibonacci_memo(n - 2) def fibonacci_iterative(n: int) -> int: """Return fib(n) iteratively. Time: O(n), Space: O(1)""" if n <= 1: return n a, b = 0, 1 for _ in range(2, n + 1): a, b = b, a + b return b
recursion.rs
use std::collections::HashMap; /// Memoized Fibonacci using a HashMap. /// Time: O(n), Space: O(n) fn fib_memo(n: u32, memo: &mut HashMap<u32, u64>) -> u64 { if let Some(&val) = memo.get(&n) { return val; } let result = match n { 0 => 0, 1 => 1, _ => fib_memo(n - 1, memo) + fib_memo(n - 2, memo), }; memo.insert(n, result); result } /// Iterative Fibonacci. Time: O(n), Space: O(1) fn fib_iterative(n: u32) -> u64 { if n == 0 { return 0; } let (mut a, mut b) = (0u64, 1u64); for _ in 2..=n { let temp = a + b; a = b; b = temp; } b }

🏘 Tower of Hanoi

Given 3 rods (Source, Auxiliary, Destination) and N disks stacked on Source (largest at bottom), move all disks to Destination following these rules:

  1. Move only one disk at a time
  2. A larger disk can never be placed on a smaller disk

Recursive Insight

To move N disks from Source to Destination using Auxiliary: Step 1: Move top (N-1) disks: Source → Auxiliary (using Dest as helper) Step 2: Move the Nth (largest) disk: Source → Destination Step 3: Move (N-1) disks: Auxiliary → Destination (using Source as helper) Base case: N == 0 → do nothing Minimum moves = 2^n - 1 n=3: 7 moves n=4: 15 moves n=10: 1,023 moves n=64: 18.4 quintillion moves (585 years at 1 billion/sec!)
TowerOfHanoi.java
// Time: O(2^n) Space: O(n) recursion depth static void hanoi(int n, char source, char auxiliary, char destination) { if (n == 0) return; // base case // Step 1: Move n-1 disks from source to auxiliary hanoi(n - 1, source, destination, auxiliary); // Step 2: Move the nth disk from source to destination System.out.println("Move disk " + n + ": " + source + " -> " + destination); // Step 3: Move n-1 disks from auxiliary to destination hanoi(n - 1, auxiliary, source, destination); } public static void main(String[] args) { hanoi(3, 'A', 'B', 'C'); // Output: 7 moves (2^3 - 1) }
recursion.cpp
// Time: O(2^n) Space: O(n) call stack void towerOfHanoi(int n, char source, char target, char auxiliary, vector<string>& moves) { if (n == 0) return; towerOfHanoi(n - 1, source, auxiliary, target, moves); moves.push_back(string(1, source) + " -> " + string(1, target)); towerOfHanoi(n - 1, auxiliary, target, source, moves); }
recursion.py
def tower_of_hanoi(n, source="A", auxiliary="B", destination="C", moves=None): """Solve Tower of Hanoi. Time: O(2^n - 1) moves, Space: O(n)""" if moves is None: moves = [] if n == 1: moves.append(f"Move disk 1 from {source} to {destination}") return moves tower_of_hanoi(n - 1, source, destination, auxiliary, moves) moves.append(f"Move disk {n} from {source} to {destination}") tower_of_hanoi(n - 1, auxiliary, source, destination, moves) return moves
recursion.rs
/// Solves Tower of Hanoi. Time: O(2^n), Space: O(n) call stack fn tower_of_hanoi(n: u32, source: &str, target: &str, auxiliary: &str, moves: &mut Vec<String>) { if n == 0 { return; } tower_of_hanoi(n - 1, source, auxiliary, target, moves); moves.push(format!("Move disk {} from {} to {}", n, source, target)); tower_of_hanoi(n - 1, auxiliary, target, source, moves); }
Exponential is Optimal

Tower of Hanoi cannot be solved in fewer than 2^n - 1 moves. It is a beautiful example of a problem where exponential time complexity is the best possible!

🎲 Recursion Patterns

Master these five patterns and you can solve the vast majority of recursive problems:

Pattern 1: Linear Recursion

One recursive call per invocation. Examples: factorial, sum, reverse string.

Pattern 2: Binary Recursion

Two recursive calls per invocation. Examples: Fibonacci, merge sort, binary search tree traversal.

Pattern 3: Tail Recursion

The recursive call is the last operation. Can be optimized by the compiler to use O(1) stack space (though Java does not optimize this).

Pattern 4: Mutual Recursion

Function A calls B, function B calls A. Example: isEven/isOdd check.

Pattern 5: Helper / Accumulator Pattern

Carry state in extra parameters to avoid redundant computation.

RecursionPatterns.java
// PATTERN 1: Linear Recursion -- Reverse a string static String reverse(String s) { if (s.isEmpty()) return ""; return reverse(s.substring(1)) + s.charAt(0); } // PATTERN 3: Tail Recursion -- Factorial with accumulator static long factTail(int n, long acc) { if (n <= 1) return acc; return factTail(n - 1, n * acc); } // PATTERN 4: Mutual Recursion static boolean isEven(int n) { if (n == 0) return true; return isOdd(n - 1); } static boolean isOdd(int n) { if (n == 0) return false; return isEven(n - 1); } // PATTERN 5: Digit Sum static int digitSum(int n) { if (n == 0) return 0; return (n % 10) + digitSum(n / 10); } // BONUS: Generate all subsets (power set) -- foundation of backtracking static void subsets(int[] arr, int idx, String current) { if (idx == arr.length) { System.out.println("{" + current + "}"); return; } subsets(arr, idx + 1, current); // Exclude String sep = current.isEmpty() ? "" : ","; subsets(arr, idx + 1, current + sep + arr[idx]); // Include }
recursion.cpp
// Reverse a string recursively string reverseStringRecursive(const string& s) { if (s.size() <= 1) return s; return reverseStringRecursive(s.substr(1)) + s[0]; } // Digit sum -- O(log10(n)) time int digitSum(int n) { if (n < 0) n = -n; if (n < 10) return n; return (n % 10) + digitSum(n / 10); } // Palindrome check (recursive) bool isPalindromeRecursive(const string& s, int left, int right) { if (left >= right) return true; if (s[left] != s[right]) return false; return isPalindromeRecursive(s, left + 1, right - 1); } // Generate all subsets using backtracking void generateSubsets(const vector<int>& nums, int index, vector<int>& current, vector<vector<int>>& result) { if (index == static_cast<int>(nums.size())) { result.push_back(current); return; } generateSubsets(nums, index + 1, current, result); // Exclude current.push_back(nums[index]); // Include generateSubsets(nums, index + 1, current, result); current.pop_back(); // backtrack }
recursion.py
def reverse_string(s: str) -> str: """Reverse a string recursively. Time: O(n), Space: O(n)""" if len(s) <= 1: return s return reverse_string(s[1:]) + s[0] def is_palindrome(s: str) -> bool: """Check if s is a palindrome recursively.""" if len(s) <= 1: return True if s[0] != s[-1]: return False return is_palindrome(s[1:-1]) def generate_subsets(nums): """Generate all subsets using backtracking. Time: O(n * 2^n)""" result = [] def backtrack(index, current): if index == len(nums): result.append(current[:]) return backtrack(index + 1, current) # Exclude current.append(nums[index]) # Include backtrack(index + 1, current) current.pop() # Backtrack backtrack(0, []) return result
recursion.rs
fn reverse_string(s: &str) -> String { if s.is_empty() { return String::new(); } let mut chars = s.chars(); let first = chars.next().unwrap(); let rest: String = chars.collect(); let mut reversed = reverse_string(&rest); reversed.push(first); reversed } fn digit_sum(n: u64) -> u64 { if n < 10 { return n; } (n % 10) + digit_sum(n / 10) } fn is_palindrome(s: &str) -> bool { let bytes = s.as_bytes(); if bytes.len() <= 1 { return true; } if bytes[0] != bytes[bytes.len() - 1] { return false; } is_palindrome(&s[1..s.len() - 1]) } /// Generate all subsets using backtracking fn generate_subsets(nums: &[i32]) -> Vec<Vec<i32>> { let mut result = Vec::new(); let mut current = Vec::new(); fn backtrack(nums: &[i32], idx: usize, cur: &mut Vec<i32>, res: &mut Vec<Vec<i32>>) { if idx == nums.len() { res.push(cur.clone()); return; } cur.push(nums[idx]); backtrack(nums, idx + 1, cur, res); cur.pop(); backtrack(nums, idx + 1, cur, res); } backtrack(nums, 0, &mut current, &mut result); result }

📚 Summary

  • Methods break problems into reusable, testable pieces. Use overloading for multiple parameter types.
  • Recursion requires a base case and a recursive case. Always check for stack depth limits.
  • Fibonacci teaches the critical lesson of overlapping subproblems -- memoization turns O(2^n) into O(n).
  • Tower of Hanoi is the classic example where exponential time is inherently optimal.
  • Recursion patterns (linear, binary, tail, mutual, backtracking) are building blocks for trees, graphs, and dynamic programming.
  • When recursion depth is a concern, consider converting to an iterative approach with an explicit stack.
