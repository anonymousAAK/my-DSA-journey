Minimum s-t Cut

WEEK 26 - WEB (JS) ADVANCED TOPICS

CONCEPT: By the max-flow / min-cut theorem the value of any maximum flow equals the capacity of the minimum s-t cut. After saturating, BFS from source in the residual graph; reachable vertices form S, the rest T, and original forward edges crossing S->T form the cut.

KEY POINTS: tag forward vs reverse edges; remember original capacity to report cut weights after residual mutation.

ALGORITHM: max-flow + residual BFS + collect crossing original edges.

JS-SPECIFIC NOTES: Spread / destructure JS makes multi-return cleaner via a single object {flow, cut}.

DRY RUN: CLRS graph -> max flow 23. Reachable set typically {0,2,4}; cut edges include (0,1)=16 and (4,3)=7 etc., total 23.

COMPLEXITY: Max-flow time + O(V+E).
