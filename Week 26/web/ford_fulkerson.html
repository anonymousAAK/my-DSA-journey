Ford-Fulkerson Maximum Flow

WEEK 26 - WEB (JS) ADVANCED TOPICS

CONCEPT: Greedy max-flow framework. Repeatedly find an augmenting s -> t path in the residual graph and push the bottleneck capacity along it. Equivalent residual-graph reasoning underpins all max-flow algorithms.

KEY POINTS: reverse edges with capacity 0 store flow; DFS path-finding; terminates by max-flow / min-cut theorem; O(E * max_flow) so prefer Edmonds-Karp / Dinic if the flow value is huge.

ALGORITHM: while DFS finds path with positive residual, push min capacity, update forward and reverse residuals, accumulate.

JS-SPECIFIC NOTES: JavaScript numbers are doubles; safe for integer capacities up to 2^53. We stub each edge as {to, cap, rev} and mutate residuals in place.

DRY RUN: CLRS graph (V=6, source 0 sink 5) -> max flow 23. Inspect the demo output.

COMPLEXITY: Time O(E * max_flow); Space O(V + E).
