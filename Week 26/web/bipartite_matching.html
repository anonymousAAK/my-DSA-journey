Bipartite Matching - Hopcroft-Karp & Kuhn

WEEK 26 - WEB (JS) ADVANCED TOPICS

CONCEPT: Maximum bipartite matching: largest set of edges with no shared endpoint. Hopcroft-Karp achieves O(E*sqrt(V)) by finding vertex-disjoint shortest augmenting paths per phase. Kuhn's algorithm is the simpler O(V*E) DFS variant.

KEY POINTS: alternating augmenting paths; matchL/matchR arrays; level-graph DFS.

ALGORITHM: while BFS finds an augmenting path of length d, run DFS for each free left vertex along level graph; flip matches.

JS-SPECIFIC NOTES: Use -1 sentinel as in Java; avoid Infinity for the dist array since we need integer ordering.

DRY RUN: 4x4 graph with 8 edges -> matching size 4 (perfect).

COMPLEXITY: Hopcroft-Karp O(E*sqrt(V)); Kuhn O(V*E).
