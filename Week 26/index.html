🌊 Ford-Fulkerson Concept

The Ford-Fulkerson method finds the maximum flow from a source s to a sink t in a flow network. The key idea: repeatedly find augmenting paths (paths from s to t with available capacity) and push flow along them until no more paths exist.

Flow Network: 10 10 s ──────> A ──────> t | | ↑ ↑ | 10 |2 |6 | | v | | 10 +──────> B ──+──────>+ Augmenting path 1: s → A → t (push 10) Augmenting path 2: s → B → t (push 10) Max Flow = 20 Key concepts: - Residual graph: shows remaining capacity + reverse edges - Augmenting path: any s→t path in residual graph - Bottleneck: minimum capacity along the path
Why Residual Edges?
Reverse edges in the residual graph allow the algorithm to "undo" flow decisions. This is essential for finding the optimal solution — without them, a greedy choice could block the true maximum flow.

🔨 Edmonds-Karp Algorithm

Edmonds-Karp is Ford-Fulkerson using BFS to find augmenting paths (shortest path in terms of edges). This guarantees O(VE²) time complexity, making it polynomial regardless of edge capacities.

NetworkFlow.java
class EdmondsKarp { int[][] capacity; int[][] flow; int n; EdmondsKarp(int n) { this.n = n; capacity = new int[n][n]; flow = new int[n][n]; } void addEdge(int u, int v, int cap) { capacity[u][v] += cap; } int maxFlow(int s, int t) { int totalFlow = 0; while (true) { // BFS to find augmenting path int[] parent = new int[n]; Arrays.fill(parent, -1); parent[s] = s; Queue<int[]> queue = new LinkedList<>(); queue.add(new int[]{s, Integer.MAX_VALUE}); while (!queue.isEmpty() && parent[t] == -1) { int[] curr = queue.poll(); int u = curr[0], pushed = curr[1]; for (int v = 0; v < n; v++) { if (parent[v] == -1 && capacity[u][v] - flow[u][v] > 0) { parent[v] = u; int newFlow = Math.min(pushed, capacity[u][v] - flow[u][v]); if (v == t) { // Augment along path totalFlow += newFlow; int cur = t; while (cur != s) { int prev = parent[cur]; flow[prev][cur] += newFlow; flow[cur][prev] -= newFlow; cur = prev; } break; } queue.add(new int[]{v, newFlow}); } } } if (parent[t] == -1) break; // no augmenting path } return totalFlow; } }

struct EdmondsKarp { int n; vector<vector<int>> capacity, flow; EdmondsKarp(int n) : n(n), capacity(n, vector<int>(n, 0)), flow(n, vector<int>(n, 0)) {} void addEdge(int u, int v, int cap) { capacity[u][v] += cap; } int maxFlow(int s, int t) { int total = 0; while (true) { vector<int> parent(n, -1); parent[s] = s; queue<pair<int,int>> q; q.push({s, INT_MAX}); while (!q.empty() && parent[t] == -1) { auto [u, pushed] = q.front(); q.pop(); for (int v = 0; v < n; ++v) { if (parent[v] == -1 && capacity[u][v] - flow[u][v] > 0) { parent[v] = u; int nf = min(pushed, capacity[u][v] - flow[u][v]); if (v == t) { total += nf; int cur = t; while (cur != s) { int prev = parent[cur]; flow[prev][cur] += nf; flow[cur][prev] -= nf; cur = prev; } break; } q.push({v, nf}); } } } if (parent[t] == -1) break; } return total; } };

from collections import deque class EdmondsKarp: def __init__(self, n): self.n = n self.capacity = [[0] * n for _ in range(n)] self.flow = [[0] * n for _ in range(n)] def add_edge(self, u, v, cap): self.capacity[u][v] += cap def max_flow(self, s, t): total = 0 while True: parent = [-1] * self.n parent[s] = s queue = deque([(s, float('inf'))]) while queue and parent[t] == -1: u, pushed = queue.popleft() for v in range(self.n): if parent[v] == -1 and self.capacity[u][v] - self.flow[u][v] > 0: parent[v] = u new_flow = min(pushed, self.capacity[u][v] - self.flow[u][v]) if v == t: total += new_flow cur = t while cur != s: prev = parent[cur] self.flow[prev][cur] += new_flow self.flow[cur][prev] -= new_flow cur = prev break queue.append((v, new_flow)) if parent[t] == -1: break return total

use std::collections::VecDeque; struct EdmondsKarp { n: usize, capacity: Vec<Vec<i64>>, flow: Vec<Vec<i64>>, } impl EdmondsKarp { fn new(n: usize) -> Self { EdmondsKarp { n, capacity: vec![vec![0; n]; n], flow: vec![vec![0; n]; n], } } fn add_edge(&mut self, u: usize, v: usize, cap: i64) { self.capacity[u][v] += cap; } fn max_flow(&mut self, s: usize, t: usize) -> i64 { let mut total = 0i64; loop { let mut parent = vec![-1i32; self.n]; parent[s] = s as i32; let mut queue = VecDeque::new(); queue.push_back((s, i64::MAX)); let mut found = false; while let Some((u, pushed)) = queue.pop_front() { if found { break; } for v in 0..self.n { if parent[v] == -1 && self.capacity[u][v] - self.flow[u][v] > 0 { parent[v] = u as i32; let nf = pushed.min(self.capacity[u][v] - self.flow[u][v]); if v == t { total += nf; let mut cur = t; while cur != s { let prev = parent[cur] as usize; self.flow[prev][cur] += nf; self.flow[cur][prev] -= nf; cur = prev; } found = true; break; } queue.push_back((v, nf)); } } } if parent[t] == -1 { break; } } total } }

✂ Max-Flow Min-Cut Theorem

The Max-Flow Min-Cut Theorem states that in any flow network, the maximum flow from source to sink equals the minimum cut capacity separating them. A min-cut is a partition of vertices into two sets (S containing source, T containing sink) that minimizes total capacity of edges crossing from S to T.

Finding the Min-Cut after running max flow: Run BFS/DFS from source in the residual graph All reachable vertices → set S All unreachable vertices → set T capacity/flow s ───(10/10)───> A ───(10/10)───> t | | ↑ (10/10) (2/2) (10/10) | v | +──────────────> B ───────────────>+ Min-Cut edges (S to T): A→t (cap 10), B→t (cap 10) Min-Cut capacity = 20 = Max Flow
Duality Insight
This duality is one of the most powerful results in combinatorial optimization. Many problems (bipartite matching, edge-disjoint paths, project selection) reduce to max-flow / min-cut.

🔗 Bipartite Matching

Given a bipartite graph with left vertices L and right vertices R, find the maximum matching — largest set of edges with no shared vertices. A simple approach uses augmenting paths via DFS.

Bipartite Graph: Maximum Matching: L R L R A ───── 1 A ===== 1 A ───── 2 B ===== 2 B ───── 2 C ===== 3 B ───── 3 C ───── 3 Matching size: 3 (maximum) C ───── 4
NetworkFlow.java
// Simple augmenting-path bipartite matching (Hungarian-style DFS) class BipartiteMatching { List<List<Integer>> adj; // adj[left] = list of right vertices int[] matchL, matchR; boolean[] visited; int leftSize, rightSize; BipartiteMatching(int left, int right) { leftSize = left; rightSize = right; adj = new ArrayList<>(); for (int i = 0; i < left; i++) adj.add(new ArrayList<>()); matchL = new int[left]; matchR = new int[right]; Arrays.fill(matchL, -1); Arrays.fill(matchR, -1); } void addEdge(int u, int v) { adj.get(u).add(v); } boolean dfs(int u) { for (int v : adj.get(u)) { if (!visited[v]) { visited[v] = true; if (matchR[v] == -1 || dfs(matchR[v])) { matchL[u] = v; matchR[v] = u; return true; } } } return false; } int maxMatching() { int result = 0; for (int u = 0; u < leftSize; u++) { visited = new boolean[rightSize]; if (dfs(u)) result++; } return result; } }

struct BipartiteMatching { int left_sz, right_sz; vector<vector<int>> adj; vector<int> matchL, matchR; vector<bool> visited; BipartiteMatching(int l, int r) : left_sz(l), right_sz(r), adj(l), matchL(l, -1), matchR(r, -1) {} void addEdge(int u, int v) { adj[u].push_back(v); } bool dfs(int u) { for (int v : adj[u]) { if (!visited[v]) { visited[v] = true; if (matchR[v] == -1 || dfs(matchR[v])) { matchL[u] = v; matchR[v] = u; return true; } } } return false; } int maxMatching() { int result = 0; for (int u = 0; u < left_sz; ++u) { visited.assign(right_sz, false); if (dfs(u)) ++result; } return result; } };

struct BipartiteMatching { adj: Vec<Vec<usize>>, match_l: Vec<i32>, match_r: Vec<i32>, left_size: usize, right_size: usize, } impl BipartiteMatching { fn new(left: usize, right: usize) -> Self { BipartiteMatching { adj: vec![vec![]; left], match_l: vec![-1; left], match_r: vec![-1; right], left_size: left, right_size: right, } } fn add_edge(&mut self, u: usize, v: usize) { self.adj[u].push(v); } fn dfs(&self, u: usize, visited: &mut Vec<bool>, ml: &mut Vec<i32>, mr: &mut Vec<i32>) -> bool { for &v in &self.adj[u] { if !visited[v] { visited[v] = true; if mr[v] == -1 || self.dfs(mr[v] as usize, visited, ml, mr) { ml[u] = v as i32; mr[v] = u as i32; return true; } } } false } fn max_matching(&mut self) -> usize { let mut result = 0; let mut ml = self.match_l.clone(); let mut mr = self.match_r.clone(); for u in 0..self.left_size { let mut visited = vec![false; self.right_size]; if self.dfs(u, &mut visited, &mut ml, &mut mr) { result += 1; } } self.match_l = ml; self.match_r = mr; result } }

⚡ Hopcroft-Karp Algorithm

Hopcroft-Karp finds maximum bipartite matching in O(E√V) time. The key improvement: instead of finding one augmenting path at a time, it finds a maximal set of shortest augmenting paths in each phase using BFS + DFS.

Algorithm Phases
1. BFS phase: Find all shortest augmenting paths (by layer) from free left vertices to free right vertices. 2. DFS phase: For each free left vertex, try to find an augmenting path along the BFS layers. 3. Repeat until BFS finds no more augmenting paths. At most O(√V) phases, each taking O(E).
NetworkFlow.java
class HopcroftKarp { List<List<Integer>> adj; int[] matchL, matchR, dist; int leftSize, rightSize; static final int NIL = -1, INF = Integer.MAX_VALUE; HopcroftKarp(int left, int right) { leftSize = left; rightSize = right; adj = new ArrayList<>(); for (int i = 0; i < left; i++) adj.add(new ArrayList<>()); matchL = new int[left]; matchR = new int[right]; dist = new int[left]; Arrays.fill(matchL, NIL); Arrays.fill(matchR, NIL); } void addEdge(int u, int v) { adj.get(u).add(v); } boolean bfs() { Queue<Integer> queue = new LinkedList<>(); for (int u = 0; u < leftSize; u++) { if (matchL[u] == NIL) { dist[u] = 0; queue.add(u); } else dist[u] = INF; } boolean found = false; while (!queue.isEmpty()) { int u = queue.poll(); for (int v : adj.get(u)) { int w = matchR[v]; // paired left vertex (or NIL) if (w == NIL) { found = true; } else if (dist[w] == INF) { dist[w] = dist[u] + 1; queue.add(w); } } } return found; } boolean dfs(int u) { for (int v : adj.get(u)) { int w = matchR[v]; if (w == NIL || (dist[w] == dist[u] + 1 && dfs(w))) { matchL[u] = v; matchR[v] = u; return true; } } dist[u] = INF; return false; } int maxMatching() { int result = 0; while (bfs()) for (int u = 0; u < leftSize; u++) if (matchL[u] == NIL && dfs(u)) result++; return result; } }

struct HopcroftKarp { int left_sz, right_sz; vector<vector<int>> adj; vector<int> matchL, matchR, dist; static constexpr int NIL = -1, INF = 1e9; HopcroftKarp(int l, int r) : left_sz(l), right_sz(r), adj(l), matchL(l, NIL), matchR(r, NIL), dist(l) {} void addEdge(int u, int v) { adj[u].push_back(v); } bool bfs() { queue<int> q; for (int u = 0; u < left_sz; ++u) { if (matchL[u] == NIL) { dist[u] = 0; q.push(u); } else dist[u] = INF; } bool found = false; while (!q.empty()) { int u = q.front(); q.pop(); for (int v : adj[u]) { int w = matchR[v]; if (w == NIL) found = true; else if (dist[w] == INF) { dist[w] = dist[u] + 1; q.push(w); } } } return found; } bool dfs(int u) { for (int v : adj[u]) { int w = matchR[v]; if (w == NIL || (dist[w] == dist[u] + 1 && dfs(w))) { matchL[u] = v; matchR[v] = u; return true; } } dist[u] = INF; return false; } int maxMatching() { int result = 0; while (bfs()) for (int u = 0; u < left_sz; ++u) if (matchL[u] == NIL && dfs(u)) ++result; return result; } };

struct HopcroftKarp { adj: Vec<Vec<usize>>, match_l: Vec<i32>, match_r: Vec<i32>, dist: Vec<i32>, left_size: usize, } impl HopcroftKarp { const NIL: i32 = -1; const INF: i32 = i32::MAX; fn new(left: usize, right: usize) -> Self { HopcroftKarp { adj: vec![vec![]; left], match_l: vec![Self::NIL; left], match_r: vec![Self::NIL; right], dist: vec![0; left], left_size: left, } } fn add_edge(&mut self, u: usize, v: usize) { self.adj[u].push(v); } fn bfs(&mut self) -> bool { let mut queue = VecDeque::new(); for u in 0..self.left_size { if self.match_l[u] == Self::NIL { self.dist[u] = 0; queue.push_back(u); } else { self.dist[u] = Self::INF; } } let mut found = false; while let Some(u) = queue.pop_front() { for &v in &self.adj[u] { let w = self.match_r[v]; if w == Self::NIL { found = true; } else if self.dist[w as usize] == Self::INF { self.dist[w as usize] = self.dist[u] + 1; queue.push_back(w as usize); } } } found } fn dfs(&mut self, u: usize) -> bool { for idx in 0..self.adj[u].len() { let v = self.adj[u][idx]; let w = self.match_r[v]; if w == Self::NIL || (self.dist[w as usize] == self.dist[u] + 1 && self.dfs(w as usize)) { self.match_l[u] = v as i32; self.match_r[v] = u as i32; return true; } } self.dist[u] = Self::INF; false } fn max_matching(&mut self) -> usize { let mut result = 0; while self.bfs() { for u in 0..self.left_size { if self.match_l[u] == Self::NIL && self.dfs(u) { result += 1; } } } result } }

💡 Applications of Flow & Matching

Network flow and bipartite matching model a surprising variety of problems. Below are two classic applications.

Assignment Problem

Given n workers and n tasks with a cost matrix, assign each worker to exactly one task to minimize total cost. This is solvable via the Hungarian Algorithm in O(n³).

Cost Matrix: Optimal Assignment: Task A Task B Task C Worker 1: 9 2 7 Worker 1 → Task B (cost 2) Worker 2: 6 4 3 Worker 2 → Task C (cost 3) Worker 3: 5 8 1 Worker 3 → Task A (cost 5) Total cost: 2 + 3 + 5 = 10 (minimum) Reduction to min-cost max-flow: Source → Workers (cap 1, cost 0) Workers → Tasks (cap 1, cost = cost[i][j]) Tasks → Sink (cap 1, cost 0)

Image Segmentation via Min-Cut

Binary image segmentation divides pixels into foreground and background. Model as a flow network: source = foreground label, sink = background label. Edge capacities encode pixel similarity and label preference.

Image grid (3x3): Flow network: +---+---+---+ | A | B | C | source (foreground) +---+---+---+ / | \ ... | D | E | F | A B C D ... +---+---+---+ \ | / ... | G | H | I | sink (background) +---+---+---+ Edge types: 1. Source → pixel: preference for foreground (data term) 2. Pixel → sink: preference for background (data term) 3. Pixel ↔ neighbor: similarity penalty (smoothness term) Min-cut separates pixels into foreground (S) and background (T)
Why This Works
The min-cut minimizes the total cost of assigning labels plus the penalty for adjacent pixels having different labels. This is a special case of submodular energy minimization, widely used in computer vision.
