🎲 Nim Game

Nim is a two-player mathematical strategy game. Players take turns removing objects from piles. The player who takes the last object wins (normal play). The key insight: a position is losing if and only if the XOR of all pile sizes is 0.

Example: Piles = [3, 4, 5] XOR: 3 ^ 4 ^ 5 = 011 ^ 100 ^ 101 = 010 = 2 (non-zero = WINNING position) Player 1 strategy: make XOR = 0 after your move Remove 2 from pile 3: [3, 4, 3] → 3 ^ 4 ^ 3 = 4 (still non-zero, bad move) Remove 2 from pile 2: [3, 2, 5] → 3 ^ 2 ^ 5 = 0 ^ 4 = 4 (wrong) Remove 2 from pile 3: [3, 4, 3] → XOR = 4 Remove 4 from pile 2: [3, 0, 5] → 3 ^ 0 ^ 5 = 6 Correct: Remove 3 from pile 3 → [3, 4, 2] → 3 ^ 4 ^ 2 = 5 (non-zero) Actually: Remove 2 from pile 1 → [1, 4, 5] → 1 ^ 4 ^ 5 = 0 (LOSING for opponent!) Key Rule: XOR = 0 means current player LOSES (if both play optimally)
Nim Strategy
If XOR of all piles is non-zero, you can always find a move to make it zero. If it's already zero, every move you make will make it non-zero, giving your opponent the advantage.
GameTheory.java
class NimGame { // Returns true if first player wins with optimal play static boolean canWin(int[] piles) { int xorSum = 0; for (int pile : piles) xorSum ^= pile; return xorSum != 0; // Non-zero XOR = first player wins } // Find optimal move: returns {pileIndex, removeCount} static int[] findMove(int[] piles) { int xorSum = 0; for (int pile : piles) xorSum ^= pile; if (xorSum == 0) return null; // Losing position for (int i = 0; i < piles.length; i++) { int target = piles[i] ^ xorSum; if (target < piles[i]) { return new int[]{i, piles[i] - target}; } } return null; } public static void main(String[] args) { int[] piles = {3, 4, 5}; System.out.println("First player wins: " + canWin(piles)); int[] move = findMove(piles); if (move != null) { System.out.println("Remove " + move[1] + " from pile " + move[0]); } } }

🧮 Sprague-Grundy Theorem

The Sprague-Grundy theorem generalizes Nim to any impartial game. Every game position has a Grundy number (nimber). A position with Grundy number 0 is losing; non-zero is winning. For combined games, XOR the Grundy numbers.

Grundy Number Calculation: G(position) = mex({G(p) : p is reachable from position}) mex = "minimum excludant" = smallest non-negative integer NOT in the set Example: Game where you can remove 1, 2, or 3 stones: G(0) = 0 (no moves, losing) G(1) = mex({G(0)}) = mex({0}) = 1 G(2) = mex({G(0), G(1)}) = mex({0, 1}) = 2 G(3) = mex({G(0), G(1), G(2)}) = mex({0, 1, 2}) = 3 G(4) = mex({G(1), G(2), G(3)}) = mex({1, 2, 3}) = 0 ← Losing! G(5) = mex({G(2), G(3), G(4)}) = mex({2, 3, 0}) = 1 Pattern: G(n) = n % 4 for this game
GameTheory.java
class SprагueGrundy { // Calculate Grundy numbers for a subtraction game // moves[] = allowed number of stones to remove static int[] grundy(int n, int[] moves) { int[] g = new int[n + 1]; for (int i = 1; i <= n; i++) { Set<Integer> reachable = new HashSet<>(); for (int m : moves) { if (i - m >= 0) reachable.add(g[i - m]); } // mex: smallest non-negative integer not in set int mex = 0; while (reachable.contains(mex)) mex++; g[i] = mex; } return g; } // Multi-pile game: XOR all Grundy numbers static boolean canWinMultiPile(int[] piles, int[] moves) { int xorSum = 0; int[] g = grundy(100, moves); // precompute up to max pile for (int pile : piles) xorSum ^= g[pile]; return xorSum != 0; } }

🔺 nCr with Pascal's Triangle

Pascal's Triangle computes binomial coefficients C(n, r) using the recurrence C(n, r) = C(n-1, r-1) + C(n-1, r). This avoids overflow issues from factorial computation and runs in O(n²) time with O(n²) space.

Pascal's Triangle: Row 0: 1 Row 1: 1 1 Row 2: 1 2 1 Row 3: 1 3 3 1 Row 4: 1 4 6 4 1 Row 5: 1 5 10 10 5 1 C(5,2) = 10 (row 5, position 2) C(4,2) = 6 (row 4, position 2) Each entry = sum of two entries above it
Combinatorics.java
class PascalTriangle { static long[][] buildPascal(int n) { long[][] C = new long[n + 1][n + 1]; for (int i = 0; i <= n; i++) { C[i][0] = 1; for (int j = 1; j <= i; j++) { C[i][j] = C[i - 1][j - 1] + C[i - 1][j]; } } return C; } // nCr with modular arithmetic for large values static long nCrMod(int n, int r, long mod) { if (r > n) return 0; long[] dp = new long[r + 1]; dp[0] = 1; for (int i = 1; i <= n; i++) { for (int j = Math.min(i, r); j > 0; j--) { dp[j] = (dp[j] + dp[j - 1]) % mod; } } return dp[r]; } public static void main(String[] args) { long[][] C = buildPascal(10); System.out.println("C(5,2) = " + C[5][2]); // 10 System.out.println("C(10,3) = " + C[10][3]); // 120 } }

🌳 Catalan Numbers

The nth Catalan number C(n) = C(2n, n) / (n+1). Catalan numbers count: valid parenthesizations, binary trees with n nodes, paths that don't cross the diagonal, triangulations of polygons, and more.

Catalan Numbers: 1, 1, 2, 5, 14, 42, 132, 429, 1430, ... C(0) = 1 C(1) = 1 C(2) = 2 C(3) = 5 C(4) = 14 Applications: Valid Parentheses: n=3 → ((())), (()()), (())(), ()(()), ()()() = 5 Binary Trees: n=3 nodes → 5 distinct shapes Ballot Problem: Paths from (0,0) to (n,n) staying below diagonal = C(n) Recurrence: C(n) = ∑ C(i) × C(n-1-i) for i=0..n-1 Formula: C(n) = C(2n, n) / (n + 1)
Combinatorics.java
class CatalanNumbers { // DP approach: O(n^2) static long[] catalanDP(int n) { long[] cat = new long[n + 1]; cat[0] = cat[1] = 1; for (int i = 2; i <= n; i++) { cat[i] = 0; for (int j = 0; j < i; j++) { cat[i] += cat[j] * cat[i - 1 - j]; } } return cat; } // Using binomial coefficient: C(n) = C(2n,n) / (n+1) static long catalanBinom(int n) { long result = 1; for (int i = 0; i < n; i++) { result = result * (2 * n - i) / (i + 1); } return result / (n + 1); } public static void main(String[] args) { long[] cat = catalanDP(10); for (int i = 0; i <= 10; i++) System.out.print(cat[i] + " "); // 1 1 2 5 14 42 132 429 1430 4862 16796 } }

🔭 Inclusion-Exclusion Principle

The Inclusion-Exclusion principle counts elements in the union of sets by alternating between adding and subtracting intersection sizes: |A ∪ B ∪ C| = |A| + |B| + |C| - |A∩B| - |A∩C| - |B∩C| + |A∩B∩C|.

Classic Application: Derangements
Count permutations where no element is in its original position. D(n) = n! × ∑(-1)^k / k! for k=0..n. Uses inclusion-exclusion on "element i is fixed" sets.
Combinatorics.java
class InclusionExclusion { // Count integers in [1, n] divisible by at least one prime in primes[] static long countDivisible(long n, int[] primes) { long count = 0; int m = primes.length; // Iterate over all non-empty subsets using bitmask for (int mask = 1; mask < (1 << m); mask++) { long product = 1; int bits = 0; for (int i = 0; i < m; i++) { if ((mask & (1 << i)) != 0) { product *= primes[i]; bits++; } } // Inclusion-exclusion: add for odd subsets, subtract for even if (bits % 2 == 1) count += n / product; else count -= n / product; } return count; } // Derangements: permutations with no fixed points static long derangement(int n) { long[] D = new long[n + 1]; D[0] = 1; D[1] = 0; for (int i = 2; i <= n; i++) D[i] = (i - 1) * (D[i - 1] + D[i - 2]); return D[n]; } }

🧮 Matrix Exponentiation

Matrix exponentiation computes the nth term of a linear recurrence in O(k³ log n) time, where k is the order of the recurrence. For Fibonacci: F(n) = [[1,1],[1,0]]^n. This technique transforms O(n) DP into O(log n).

Fibonacci via Matrix Exponentiation: | F(n+1) | | 1 1 |^n | F(1) | | 1 1 |^n | 1 | | | = | | × | | = | | × | | | F(n) | | 1 0 | | F(0) | | 1 0 | | 0 | Use fast exponentiation (repeated squaring) on matrices! Example: F(10) = 55 M^1 = [[1,1],[1,0]] M^2 = [[2,1],[1,1]] M^4 = [[5,3],[3,2]] M^8 = [[34,21],[21,13]] M^10 = M^8 × M^2 = [[89,55],[55,34]] F(10) = 55
Combinatorics.java
class MatrixExpo { static final long MOD = 1_000_000_007; static long[][] multiply(long[][] A, long[][] B) { int n = A.length; long[][] C = new long[n][n]; for (int i = 0; i < n; i++) for (int k = 0; k < n; k++) if (A[i][k] != 0) for (int j = 0; j < n; j++) C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % MOD; return C; } static long[][] matPow(long[][] M, long p) { int n = M.length; long[][] result = new long[n][n]; for (int i = 0; i < n; i++) result[i][i] = 1; // Identity while (p > 0) { if ((p & 1) == 1) result = multiply(result, M); M = multiply(M, M); p >>= 1; } return result; } // Fibonacci in O(log n) static long fibonacci(long n) { if (n <= 1) return n; long[][] M = {{1, 1}, {1, 0}}; long[][] result = matPow(M, n - 1); return result[0][0]; // F(n) } }
